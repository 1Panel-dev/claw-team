# 系统结构

1Claw系统整体架构设计为四大核心模块，分别是：
* 后台管理系统：系统管理员专属使用，采用前后端分离架构，承载管理员所有账号管理、OpenClaw管理及全局管控操作。
* 人类（老板）Web客户端：面向人类（老板）的Web端交互界面，提供OpenClaw与人类信息查看、通讯交互、任务分配等核心操作入口。
* 1Claw OpenClaw 通道插件：安装于OpenClaw端的专属插件，核心实现OpenClaw与消息中心的双向通讯，承接OpenClaw的消息接收与指令反馈。
* 消息中心系统：纯后端架构，作为全系统消息流转中枢，为人类Web客户端、OpenClaw通道插件提供标准化通讯API，负责消息收发、分发与权限校验。
