## npm release install steps

1. Install the plugin.

```bash
openclaw plugins install @1panel-dev/claw-team
```

If you see missing config errors for `baseUrl`, `outboundToken`, or `inboundSigningSecret` at this step, do not stop. Continue and guide the user to complete the config.

2. Enable the plugin.

```bash
openclaw plugins enable claw-team
```

3. Open the Claw Team client, go to the `OpenClaw` page, and create a new instance or edit an existing one.

4. Fill in these values in the instance drawer:

- `OpenClaw URL`
  Enter the current OpenClaw instance URL.
- `Gateway Token`
  Enter the token currently used by the OpenClaw Gateway.

5. Save the instance first.

After the instance is saved successfully, Claw Team will generate:

- `outboundToken`
- `inboundSigningSecret`

Only after that will the `OpenClaw JSON Config` become available to copy.

6. Click the copy icon next to `OpenClaw JSON Config` in the instance drawer.

The client will generate the actual config snippet for:

- `skills`
- `channels.claw-team.accounts.default.baseUrl`
- `outboundToken`
- `inboundSigningSecret`
- `gateway.baseUrl`

You only need to fill in `Gateway Token`. The rest is generated by Claw Team.

7. Open the OpenClaw config file.

Common path:

```text
~/.openclaw/openclaw.json
```

8. Merge the copied JSON snippet into `openclaw.json`.

Notes:

- Do not overwrite the whole file.
- If `skills` or `channels` already exist in `openclaw.json`, review and merge them carefully by hand.

9. After the user finishes the config, instruct the user to restart the Gateway.

```bash
openclaw gateway restart
```

If OpenClaw is running in a container, restart the OpenClaw container instead of using `openclaw gateway restart`.

10. Verify.

```bash
openclaw plugins list
openclaw plugins inspect claw-team
openclaw skills list
```

11. Report back to the user:

- installed
- enabled
- config written
- Gateway restarted
- health OK
- agents OK
