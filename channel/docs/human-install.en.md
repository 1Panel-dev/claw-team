# Human Install

[中文版](./human-install.zh-CN.md)

## npm release install

1. Install the plugin.

```bash
openclaw plugins install @1panel-dev/claw-team
```

If this step returns `429 Rate Limit Exceeded`, ClawHub is rate-limiting. Do not stop here. Switch to manual npm installation instead:

```bash
cd /tmp
npm pack @1panel-dev/claw-team
mkdir -p /tmp/claw-team-pkg
cd /tmp/claw-team-pkg
tar xzf /tmp/1panel-dev-claw-team-1.0.1.tgz
```

This extracts the package into `/tmp/claw-team-pkg/package/`.

Then copy it into the OpenClaw extensions directory:

```bash
cp -r /tmp/claw-team-pkg/package /home/node/.openclaw/extensions/claw-team
```

If `/home/node/.openclaw/extensions/claw-team` already exists, do not delete it blindly. Check the directory first. If it contains local changes, back it up before replacing it. For example:

```bash
ls -la /home/node/.openclaw/extensions/claw-team
mv /home/node/.openclaw/extensions/claw-team /home/node/.openclaw/extensions/claw-team.bak.$(date +%Y%m%d-%H%M%S)
cp -r /tmp/claw-team-pkg/package /home/node/.openclaw/extensions/claw-team
```

Then install plugin dependencies:

```bash
cd /home/node/.openclaw/extensions/claw-team
npm install --omit=dev
```

If you see missing config errors for `baseUrl`, `outboundToken`, or `inboundSigningSecret` at this step, do not stop. Continue to the config step.

2. Enable the plugin.

```bash
openclaw plugins enable claw-team
```

3. Open the Claw Team client, go to the `OpenClaw` page, and create a new instance or edit an existing one.

4. Fill in these values in the instance drawer:

- `OpenClaw URL`
  Enter the current OpenClaw instance URL.
- `Gateway Token`
  Enter the token currently used by the OpenClaw Gateway.

5. Save the instance first.

After the instance is saved successfully, Claw Team will generate:

- `outboundToken`
- `inboundSigningSecret`

Only after that will the `OpenClaw JSON Config` become available to copy.

6. Click the copy icon next to `OpenClaw JSON Config` in the instance drawer.

The client will generate the full OpenClaw config snippet, including:

- `plugins.allow`
- `plugins.entries.claw-team`
- `skills`
- `channels.claw-team.accounts.default.baseUrl`
- `outboundToken`
- `inboundSigningSecret`
- `gateway.baseUrl`
- `webchatMirror.includeIntermediateMessages`

You only need to fill in `Gateway Token`. The rest is generated by Claw Team.

7. Open the OpenClaw config file.

Common path:

```text
~/.openclaw/openclaw.json
```

8. Merge the copied OpenClaw JSON Config into `openclaw.json`.

Notes:

- Do not overwrite the whole file.
- If `plugins`, `skills`, or `channels` already exist in `openclaw.json`, review and merge them carefully by hand.
- `plugins.allow` does not support wildcards like `["*"]`.
- You must explicitly list `claw-team`.
- Unknown plugin IDs will fail config validation.

9. Restart the Gateway.

```bash
openclaw gateway restart
```

If OpenClaw is running in a container, restart the OpenClaw container instead of using `openclaw gateway restart`, or send `USR1` to the Gateway process.

10. Verify the installation.

```bash
openclaw plugins list
openclaw plugins inspect claw-team
openclaw skills list
```

You should see:

- `claw-team` status is `loaded`
- `ct-chat` skill status is `ready`
