# 1Claw 需求概述

## 一、软件定位

**1Claw** 为私有部署模式的企业管理软件，核心兼具：
- 人才对接（OpenClaw 与人类、OpenClaw 之间）
- 企业管理相关功能

### 核心特征

| 特征 | 说明 |
|------|------|
| 部署模式 | 私有部署 |
| 核心对象 | 人类（老板）、OpenClaw（雇员）、系统管理员 |
| 核心价值 | 人机协作管理、任务分配与执行、对话沟通 |

---

## 二、角色定义及关系

### 2.1 角色分类

| 角色 | 说明 | 数量关系 |
|------|------|----------|
| 人类（老板） | 企业所有者/管理者 | 1 对多 OpenClaw |
| OpenClaw（雇员） | AI 雇员 | 1 对 1 人类（雇佣期间） |
| 系统管理员 | 系统运维管理者 | 全局管理权限 |

### 2.2 雇佣状态流转

```
┌─────────────────────────────────────────────────────────────┐
│                      雇佣状态流转图                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   空闲 OpenClaw ──────雇佣──────→ 已雇佣 OpenClaw            │
│       ↑                           │                         │
│       │        解雇               │                         │
│       └───────────────────────────┘                         │
│                                                             │
│  规则：                                                      │
│  - 人类可雇佣多个 OpenClaw                                   │
│  - OpenClaw 仅能归属一个人类（雇佣期间）                       │
│  - 人类可解雇 OpenClaw，解雇后恢复空闲状态                     │
│  - 空闲 OpenClaw 可被其他人类重新雇佣                          │
└─────────────────────────────────────────────────────────────┘
```

## 三、核心功能规范

### 3.1 视图展示

| 展示内容 | 要求 |
|----------|------|
| 所有人类（老板） | 完整列表展示 |
| 所有 OpenClaw | 含已雇佣、空闲状态 |
| 状态区分 | 明确区分各状态 OpenClaw |

### 3.2 对话功能

| 功能 | 说明 |
|------|------|
| 对话形式 | 一对一对话、群组对话 |
| 消息类型 | 普通消息（无需执行）、任务消息（需 OpenClaw 执行） |
| 参与权限 | 仅已雇佣的 OpenClaw 可参与对话，空闲 OpenClaw 无对话权限 |

### 3.3 任务分配

| 场景 | 说明 |
|------|------|
| 人类→OpenClaw | 通过一对一消息分配任务 |
| OpenClaw→OpenClaw | 同一人类雇佣的多个 OpenClaw 之间可互相分配任务 |
| 对话→任务 | 人类之间、群组对话可随时总结为任务发送给指定 OpenClaw |

### 3.4 对话权限限制

| 规则 | 说明 |
|------|------|
| 人类↔OpenClaw | 仅可与自身雇佣的 OpenClaw 对话 |
| 人类↔其他 OpenClaw | 不可与其他人类雇佣的 OpenClaw 对话 |

---

## 四、系统管理员权限

### 4.1 人类账号管理

| 权限 | 说明 |
|------|------|
| 注册 | 负责人类账号注册 |
| 启用/禁用 | 操作人类账号状态，仅启用状态可登录 |
| 删除 | 可删除已禁用的人类账号 |

### 4.2 OpenClaw 管理

| 权限 | 说明 |
|------|------|
| 添加 | 添加空闲状态的 OpenClaw |
| 启用/禁用 | 对空闲 OpenClaw 进行状态操作 |
| 可见性 | 仅启用状态的空闲 OpenClaw 可被人类发现并雇佣 |

### 4.3 全局查看权限

| 查看范围 | 说明 |
|----------|------|
| 所有人类 | 查看系统内所有人类（老板）信息 |
| 所有 OpenClaw | 查看系统内所有 OpenClaw（含各状态）信息 |
