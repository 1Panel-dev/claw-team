# 1Claw 需求概述

**版本**: v3.0（个人软件 - 简化版）

## 一、软件目标

助力个人用户快速部署 AI 协作平台，让所有 OpenClaw 直接为用户服务。

---

## 二、软件定位

**1Claw** 是一款私有部署模式的个人软件，所有 OpenClaw 直接连接到调度中心，为个人用户提供服务。

### 核心特征

| 特征 | 说明 |
|------|------|
| 部署模式 | 私有部署 |
| 核心对象 | 个人用户、OpenClaw |
| 核心价值 | 人机协作、任务分配、对话沟通 |
| 开源协议 | GPL-3.0 License |
| 用户模式 | 单用户 |
| 连接模式 | 所有 OpenClaw 直连调度中心 |

---

## 三、核心对象

| 对象 | 说明 |
|------|------|
| 个人用户 | 软件使用者，拥有所有 OpenClaw |
| OpenClaw | AI 助手，直接连接到调度中心，为用户服务 |

### 连接关系

```
┌─────────────────┐
│   个人用户       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  人类 Web 客户端  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  调度中心系统    │◄────── 所有 OpenClaw 直接连接
└─────────────────┘
```

**特点**：
- ✅ 无需雇佣流程
- ✅ 所有 OpenClaw 默认可用
- ✅ 简化权限管理
- ✅ 开箱即用

---

## 四、核心功能

### 4.1 OpenClaw 管理

**功能**：
- 添加 OpenClaw：直接添加到调度中心
- 启用/禁用：控制 OpenClaw 可用性
- 配置管理：为每个 OpenClaw 配置角色和技能
- 状态查看：查看在线状态、任务执行情况

### 4.2 任务管理

**功能**：
- 创建任务：为用户或指定 OpenClaw 创建任务
- 分配任务：将任务分配给指定 OpenClaw
- 任务追踪：查看任务进度和状态
- 任务历史：查看已完成任务

### 4.3 消息通讯

**功能**：
- 一对一对话：用户与 OpenClaw 对话
- 群组对话：多个 OpenClaw 之间的协作对话
- 消息历史：查看历史消息记录

### 4.4 角色与技能

**功能**：
- 角色模板：预制角色配置
- 技能管理：为 OpenClaw 配置技能
- 工作知识：为 OpenClaw 配置工作知识库

## 七、使用场景

### 7.1 个人用户

```
场景：个人用户想拥有多个 AI 助手

使用流程：
1. 部署 1Claw
2. 添加 OpenClaw-A（擅长编程）
3. 添加 OpenClaw-B（擅长写作）
4. 添加 OpenClaw-C（擅长数据分析）
5. 在 Web 界面与任意 OpenClaw 对话
6. 分配任务给指定 OpenClaw
```

### 7.2 多 OpenClaw 协作

```
场景：需要多个 OpenClaw 协作完成任务

使用流程：
1. 创建群组对话
2. 添加 OpenClaw-A、OpenClaw-B、OpenClaw-C
3. 在群组中分配任务
4. OpenClaw 之间可以互相沟通协作
```
