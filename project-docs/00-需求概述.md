# 1Claw 需求概述

## 一、软件目标

助力个人用户快速部署 AI 协作平台，通过多 Agent 协作完成复杂任务。

---

## 二、软件定位

**1Claw** 是一款私有部署模式的个人软件，支持个人用户管理多个 OpenClaw，每个 OpenClaw 可配置多个 Agent（身份/员工），实现多角色 AI 协作。

### 核心特征

| 特征 | 说明 |
|------|------|
| 部署模式 | 私有部署 |
| 核心对象 | 个人用户、OpenClaw、Agent（身份/员工） |
| 核心价值 | 人机协作、多 Agent 协作、任务分配 |
| 开源协议 | GPL-3.0 License |
| 用户模式 | 单用户 |
| 连接模式 | 所有 OpenClaw 直连调度中心 |

---

## 三、核心对象

| 对象 | 说明 |
|------|------|
| 个人用户 | 软件使用者，拥有所有 OpenClaw 和 Agent |
| OpenClaw | AI 连接实例，可承载多个 Agent |
| Agent（身份/员工） | AI 角色实例，拥有独立记忆和配置，执行具体任务 |

### 层次关系

```
┌─────────────────┐
│   个人用户       │
└────────┬────────┘
         │
         ▼
┌─────────────────────────────────────┐
│  OpenClaw #1                        │
│  ├─ Agent A [程序员]                 │
│  ├─ Agent B [作家]                  │
│  └─ Agent C [数据分析师]             │
└─────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│  OpenClaw #2                        │
│  ├─ Agent D [翻译]                  │
│  └─ Agent E [设计师]                 │
└─────────────────────────────────────┘
```

### Agent（身份/员工）

**定义**：
- Agent 是 AI 的具体身份/角色
- 每个 Agent 有独立的配置和记忆
- Agent 之间相互隔离
- 一个 OpenClaw 可承载多个 Agent
- 未配置 Agent 时使用 OpenClaw 默认 Agent

**示例**：
```
OpenClaw #1
├── Agent: 程序员 - 负责编程任务
├── Agent: 作家 - 负责写作任务
└── Agent: 数据分析师 - 负责数据分析任务
```

## 四、核心功能

### 4.1 OpenClaw 管理

**功能**：
- 添加 OpenClaw：连接到调度中心
- 启用/禁用：控制 OpenClaw 可用性
- 配置管理：为 OpenClaw 配置多个 Agent

### 4.2 Agent（身份/员工）管理

**功能**：
- 创建 Agent：为 OpenClaw 添加新身份
- 配置 Agent：设置角色、技能、工作知识
- 启用/禁用：控制 Agent 可用性
- 独立记忆：每个 Agent 拥有独立记忆

### 4.3 对话通讯

**功能**：
- **一对一对话**：用户 ↔ 某个 Agent
- **群组对话**：用户 + 多个 Agent（可跨 OpenClaw）
- **Agent 间协作**：多个 Agent 在群组中协作
- 消息历史：查看历史消息记录

**群组规则**：
- 群组必须由用户创建
- 用户必须在群组中
- 群组可包含多个 Agent（可跨 OpenClaw）

### 4.4 任务管理

**功能**：
- 创建任务：为用户或指定 Agent 创建任务
- 分配任务：将任务分配给具体的 Agent（某个 OpenClaw 上的身份）
- 任务追踪：查看任务进度和状态
- 任务历史：查看已完成任务

**任务分配规则**：
- 任务必须分配给具体的 Agent
- 任务列表公开可见
- 执行时必须有明确的执行者（Agent）

### 4.5 角色与技能

**功能**：
- 角色模板：预制角色配置（程序员、作家、设计师等）
- 技能管理：为 Agent 配置技能
- 工作知识：为 Agent 配置工作知识库

## 五、使用场景

### 5.1 单 OpenClaw 多 Agent 协作

```
场景：个人用户需要一个 AI 团队完成项目

配置：
1. 添加 OpenClaw #1
2. 创建 Agent A [程序员] - 负责编码
3. 创建 Agent B [产品经理] - 负责需求
4. 创建 Agent C [测试工程师] - 负责测试

使用流程：
1. 创建群组对话：用户 + Agent A + Agent B + Agent C
2. 在群组中讨论项目
3. 分配任务：
   - 任务 1 → Agent A（编码）
   - 任务 2 → Agent B（需求文档）
   - 任务 3 → Agent C（测试用例）
4. 各 Agent 协作完成项目 ✅
```

### 5.2 多 OpenClaw 跨实例协作

```
场景：需要不同专长的 AI 协作

配置：
1. OpenClaw #1（擅长技术）
   - Agent A [后端工程师]
   - Agent B [前端工程师]
2. OpenClaw #2（擅长创意）
   - Agent C [UI 设计师]
   - Agent D [文案策划]

使用流程：
1. 创建群组对话：用户 + Agent A + Agent B + Agent C + Agent D
2. 跨 OpenClaw 协作讨论
3. 分配任务给具体 Agent
4. 完成跨领域协作项目 ✅
```

### 5.3 一对一专业咨询

```
场景：需要特定角色的专业建议

使用流程：
1. 选择 Agent [数据分析师]
2. 发起一对一对话
3. 上传数据，请求分析
4. 获得专业分析报告 ✅
```

## 六、技术特点

| 特点 | 说明 |
|------|------|
| 私有部署 | 数据完全本地存储 |
| 单用户 | 无需多用户隔离 |
| 多 Agent | 一个 OpenClaw 可承载多个 Agent |
| 独立记忆 | 每个 Agent 拥有独立记忆 |
| Agent 隔离 | Agent 之间相互隔离 |
| 群组协作 | 支持跨 OpenClaw 的 Agent 协作 |
| SQLite | 默认数据库，零配置 |
| 开源 | GPL-3.0 协议 |
