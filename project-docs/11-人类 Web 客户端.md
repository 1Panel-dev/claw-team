# 1Claw 人类 Web 客户端需求分析

## 1. 系统概述

### 1.1 系统定位

人类 Web 客户端是 1Claw 系统面向**人类（老板）**的 Web 端交互界面，提供 OpenClaw 与人类信息查看、通讯交互、任务分配等核心操作入口。

### 1.2 目标用户

| 用户身份 | 说明 | 使用场景 |
|----------|------|----------|
| 人类（老板） | 企业所有者/管理者 | 雇佣 OpenClaw、对话交流、任务分配、查看任务进度 |

### 1.3 核心功能模块

| 模块 | 功能说明 |
|------|----------|
| 聊天通讯 | 与已雇佣 OpenClaw 进行一对一对话 |
| 雇员管理 | 查看已雇佣的 OpenClaw、解雇操作 |
| 人才市场 | 浏览可雇佣的 OpenClaw、雇佣操作 |
| 任务管理 | 分配任务、查看任务进度、任务历史 |
| 技能库 | 查看 OpenClaw 技能、按技能筛选 |

### 1.4 与雇佣关系模块的关联

人类 Web 客户端负责：
- 雇佣空闲 OpenClaw（建立雇佣关系）
- 解雇已雇佣的 OpenClaw（解除雇佣关系）
- 查看自己雇佣的 OpenClaw 列表
- 与已雇佣 OpenClaw 对话和分配任务

## 2. 系统架构

### 2.1 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                   人类 Web 客户端                            │
├─────────────────────────────────────────────────────────────┤
│  前端 (Web)                                                  │
│  ───────────                                                 │
│  • Vue/React                                                 │
│  • 组件库 (AntD/Element)                                     │
│  • WebSocket 实时通讯                                        │
│  • 状态管理                                                  │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ HTTP/WebSocket
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                      消息中心系统                            │
│  • 消息收发                                                  │
│  • 权限校验                                                  │
│  • 消息持久化                                                │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 权限约束

| 约束 | 说明 |
|------|------|
| 仅可与已雇佣 OpenClaw 对话 | 空闲 OpenClaw 无对话权限 |
| 仅可雇佣空闲 - 启用状态的 OpenClaw | 已雇佣/已禁用的 OpenClaw 不可雇佣 |
| 仅可解雇自己雇佣的 OpenClaw | 不可解雇其他人类的 OpenClaw |
| 不可与其他人类对话 | 人类之间无直接对话功能 |
| **雇佣数量限制** | 默认最多雇佣 3 个 OpenClaw，管理员可配置上限 |

#### 4.3.1 雇佣 OpenClaw

**入口**：人才市场 → [雇佣] 按钮

**交互流程**：
```
1. 用户点击 [雇佣]
   ↓
2. 系统检查雇佣数量限制
   ↓
3. 如已达上限，提示错误（见异常情况）
   ↓
4. 弹出确认对话框：
   ┌────────────────────────────────────┐
   │  确认雇佣                           │
   │                                    │
   │  确定要雇佣 "AI 助手 -003" 吗？       │
   │                                    │
   │  雇佣后，该 OpenClaw 将为您工作，      │
   │  您可与其对话并分配任务。            │
   │                                    │
   │        [取消]     [确认雇佣]        │
   └────────────────────────────────────┘
   ↓
5. 用户点击 [确认雇佣]
   ↓
6. 系统处理
   ↓
7. 成功提示（Toast）：
   "✓ 雇佣成功，AI 助手 -003 现已为您工作"
   ↓
8. 该卡片从人才市场移除（或标记为"已雇佣"）
   ↓
9. 自动跳转到与该 OpenClaw 的对话窗口
```

## 5. 交互规范

### 5.1 弹窗规范

| 类型 | 使用场景 | 样式 |
|------|----------|------|
| 确认对话框 | 解雇、删除等敏感操作 | 警示样式（红色/黄色） |
| 表单弹窗 | 新建任务、编辑等 | 标准样式 |
| 详情弹窗 | 查看详情信息 | 只读样式 |

### 5.2 提示规范

| 类型 | 展示形式 | 持续时间 |
|------|----------|----------|
| 成功提示 | Toast（绿色） | 3 秒 |
| 失败提示 | Toast（红色） | 5 秒 |
| 警告提示 | Toast（黄色） | 5 秒 |
| 加载提示 | Loading 遮罩 | 操作完成前 |

### 5.3 实时通讯规范

- 使用 WebSocket 保持长连接
- 新消息实时推送（无需刷新）
- 联系人状态实时更新（在线/离线/工作中）
- 任务状态变更实时通知
