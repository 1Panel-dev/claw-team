# 1Claw 人类 Web 客户端需求分析

---

## 1. 系统概述

### 1.1 系统定位

人类 Web 客户端是 1Claw 系统面向**个人用户**的 Web 端交互界面，提供 OpenClaw 管理、Agent（身份/员工）管理、通讯交互、任务分配等核心操作入口。

### 1.2 目标用户

| 用户身份 | 说明 | 使用场景 |
|----------|------|----------|
| 个人用户 | 软件使用者 | 管理 OpenClaw 和 Agent、对话交流、任务分配 |

### 1.3 核心功能模块

| 模块 | 功能说明 |
|------|----------|
| OpenClaw 管理 | 添加、启用/禁用、配置 OpenClaw |
| Agent 管理 | 为 OpenClaw 创建、配置、管理多个 Agent（身份/员工） |
| 聊天通讯 | 与 Agent 进行一对一对话、群组对话 |
| 任务管理 | 分配任务给具体 Agent、查看任务进度、任务历史 |

## 4. OpenClaw 管理

### 4.1 OpenClaw 列表

**显示内容**：
- OpenClaw 名称
- OpenClaw 状态（在线/离线）
- OpenClaw 下的 Agent 列表
- 每个 Agent 的状态（启用/禁用）

**状态标识**：
- 🟢 在线 - 已连接
- ⚪ 离线 - 未连接
- ✅ 启用 - 可接受任务
- ⏸️ 禁用 - 不可用

### 4.2 添加 OpenClaw

**流程**：
```
1. 点击"添加 OpenClaw"
   ↓
2. 填写 OpenClaw 信息
   - 名称
   - 描述（可选）
   ↓
3. 点击"添加"
   ↓
4. OpenClaw 出现在列表中
   ↓
5. OpenClaw 启动后自动连接 ✅
```

### 4.3 启用/禁用 OpenClaw

**启用**：
- OpenClaw 可接受连接
- OpenClaw 下的 Agent 可被使用

**禁用**：
- OpenClaw 不可接受连接
- OpenClaw 下的所有 Agent 不可用

## 5. Agent（身份/员工）管理

### 5.1 Agent 列表

**显示内容**：
- Agent 名称/身份
- Agent 所属 OpenClaw
- Agent 角色（程序员、作家、设计师等）
- Agent 状态（启用/禁用）
- 最后活跃时间

**示例**：
```
OpenClaw #1
├── Agent: 程序员 [✅ 启用] - 最后活跃：10 分钟前
├── Agent: 作家 [✅ 启用] - 最后活跃：1 小时前
└── Agent: 数据分析师 [⏸️ 禁用] - 最后活跃：昨天
```

### 5.2 创建 Agent

**流程**：
```
1. 选择 OpenClaw
   ↓
2. 点击"创建 Agent"
   ↓
3. 填写 Agent 信息
   - 名称（如：程序员 -001）
   - 角色（选择预设或自定义）
   - 技能配置
   - 工作知识库（可选）
   ↓
4. 点击"创建"
   ↓
5. Agent 创建成功 ✅
```

**配置项**：
- 名称（必填）
- 角色模板（可选）
- 技能配置
- 工作知识库
- 系统提示词（定义身份和行为）

### 5.3 Agent 独立记忆

**特点**：
- 每个 Agent 拥有独立记忆
- Agent 之间记忆不共享
- 记忆与 Agent 绑定，不随 OpenClaw 变化

**记忆内容**：
- 对话历史
- 任务执行记录
- 用户偏好
- 工作知识

### 5.4 启用/禁用 Agent

**启用**：
- Agent 可接受对话
- Agent 可接受任务
- 在联系人列表中显示

**禁用**：
- Agent 不可接受对话
- Agent 不可接受任务
- 在联系人列表中标记为"已禁用"

## 6. 聊天通讯

### 6.1 一对一对话

**对话对象**：
- 任意已启用的 Agent

**功能**：
- 发送文本消息
- 发送文件
- 查看历史消息
- 标记已读/未读

**示例**：
```
用户 ↔ Agent[程序员]
用户 ↔ Agent[作家]
用户 ↔ Agent[设计师]
```

### 6.2 群组对话

**创建群组**：
1. 点击"创建群组"
2. 选择多个 Agent（可跨 OpenClaw）
3. 设置群组名称
4. 创建成功

**规则**：
- 群组必须由用户创建
- 用户必须在群组中
- 群组可包含多个 Agent
- Agent 可来自不同 OpenClaw

**功能**：
- 多 Agent 协作
- 群组消息广播
- 任务分配给群组中的 Agent

**示例**：
```
群组：项目开发群
成员：用户 + Agent[程序员] + Agent[设计师] + Agent[产品经理]
```

## 7. 任务管理

### 7.1 创建任务

**流程**：
```
1. 点击"创建任务"
   ↓
2. 填写任务信息
   - 标题
   - 描述
   - 截止时间
   - 优先级
   - 分配给（可选具体 Agent）
   ↓
3. 点击"创建"
   ↓
4. 任务创建成功 ✅
```

**任务信息**：
- 标题（必填）
- 描述（必填）
- 截止时间（必填）
- 优先级（低/中/高/紧急）
- 分配给（可选具体 Agent，可稍后分配）

### 7.2 分配任务

**分配规则**：
- 任务必须分配给具体的 Agent
- 不能分配给 OpenClaw 整体
- 可跨 OpenClaw 分配

**分配方式**：
- 创建任务时直接分配给 Agent
- 创建后分配给指定 Agent
- 在群组中分配给群组中的 Agent

**示例**：
```
任务：开发登录页面
分配给：Agent[前端工程师] (OpenClaw #1)

任务：设计 UI 界面
分配给：Agent[UI 设计师] (OpenClaw #2)
```

### 7.3 任务状态

| 状态 | 说明 |
|------|------|
| new | 新建 |
| in_progress | 进行中 |
| completed | 已完成 |
| accepted | 已验收 |
| rejected | 已驳回 |

## 8. API 接口

### 8.1 OpenClaw 管理 API

```
GET    /api/openclaws              # 获取所有 OpenClaw
POST   /api/openclaws              # 添加 OpenClaw
PUT    /api/openclaws/:id          # 更新 OpenClaw 配置
DELETE /api/openclaws/:id          # 删除 OpenClaw
POST   /api/openclaws/:id/enable   # 启用 OpenClaw
POST   /api/openclaws/:id/disable  # 禁用 OpenClaw
```

### 8.2 Agent 管理 API

```
GET    /api/openclaws/:id/agents        # 获取 OpenClaw 下的所有 Agent
POST   /api/openclaws/:id/agents        # 创建 Agent
GET    /api/agents/:id                  # 获取 Agent 详情
PUT    /api/agents/:id                  # 更新 Agent 配置
DELETE /api/agents/:id                  # 删除 Agent
POST   /api/agents/:id/enable           # 启用 Agent
POST   /api/agents/:id/disable          # 禁用 Agent
GET    /api/agents/:id/memory           # 获取 Agent 记忆
```

### 8.3 消息 API

```
POST   /api/messages/send          # 发送消息
GET    /api/messages/history       # 获取消息历史
GET    /api/messages/unread        # 获取未读消息
POST   /api/messages/read          # 标记已读
```

### 8.4 任务 API

```
POST   /api/tasks                  # 创建任务
GET    /api/tasks                  # 获取任务列表
GET    /api/tasks/:id              # 获取任务详情
PUT    /api/tasks/:id              # 更新任务
PUT    /api/tasks/:id/assign       # 分配任务给具体 Agent
POST   /api/tasks/:id/complete     # 完成任务
POST   /api/tasks/:id/accept       # 验收任务
```

### 8.5 群组 API

```
POST   /api/groups                 # 创建群组
GET    /api/groups                 # 获取群组列表
GET    /api/groups/:id             # 获取群组详情
PUT    /api/groups/:id/members     # 更新群组成员
DELETE /api/groups/:id             # 删除群组
```
