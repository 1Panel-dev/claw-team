# 1Claw 任务管理详细需求

---

## 1. 系统概述

### 1.1 系统定位

任务管理是 1Claw 的核心功能之一，支持用户创建任务、分配给具体的 Agent（某个 OpenClaw 上的身份/员工）、追踪任务进度和查看任务结果。

### 1.2 任务生命周期

```
创建 → in_progress（正在执行）
              ↓
         ┌────┴────┐
         ▼         ▼
    completed  terminated
    (已完成)    (已终止)
```

---

## 2. 任务状态

### 2.1 状态定义

| 状态 | 说明 | 触发时机 |
|------|------|---------|
| `in_progress` | 正在执行 | 任务创建后默认状态 |
| `completed` | 已完成 | Agent 完成任务，用户确认 |
| `terminated` | 已终止 | 用户或系统终止任务 |

### 2.2 状态流转

**正常流程**：
```
创建 → in_progress → completed
```

**终止流程**：
```
创建 → in_progress → terminated
```

### 2.3 状态操作

| 当前状态 | 可执行操作 | 结果状态 |
|---------|-----------|---------|
| `in_progress` | 完成任务 | `completed` |
| `in_progress` | 终止任务 | `terminated` |
| `completed` | 无（结束） | - |
| `terminated` | 无（结束） | - |

---

## 3. 任务创建

### 3.1 创建规则

**必须字段**：
- 标题（最多 200 字符）
- 描述（支持 Markdown）
- **执行者（必须指定具体 Agent）**

**可选字段**：
- 优先级（低/中/高/紧急）
- 标签
- 附件

**自动字段**：
- 状态：`in_progress`（创建后默认）
- 开始时间：`started_at` = 创建时间

### 3.2 任务数据结构

```json
{
  "task_id": "task_001",
  "title": "开发登录页面",
  "description": "开发一个用户登录页面...",
  "priority": "high",
  "status": "in_progress",
  "assignee": {
    "openclaw_id": "oc_001",
    "agent_id": "agent_A"
  },
  "started_at": "2026-03-18T05:00:00Z",
  "ended_at": null,
  "created_by": "user",
  "created_at": "2026-03-18T05:00:00Z"
}
```

### 3.3 分配规则

| 规则 | 说明 |
|------|------|
| 必须分配给 Agent | 创建任务时必须指定具体 Agent |
| Agent 必须启用 | 禁用的 Agent 不可分配 |
| 可跨 OpenClaw 分配 | 可分配给任意 OpenClaw 上的 Agent |

**示例**：
```
✅ 正确：
{
  "title": "开发登录页面",
  "assignee": {
    "openclaw_id": "oc_001",
    "agent_id": "agent_A"
  }
}

❌ 错误：
{
  "title": "开发登录页面",
  "assignee": null  # 不允许
}
```

## 4. 任务执行

### 4.1 Agent 操作

| 操作 | 说明 |
|------|------|
| 执行任务 | 状态保持 `in_progress` |
| 完成任务 | 状态：`in_progress` → `completed` |
| 请求帮助 | 在群组中发送消息给用户 |

### 4.2 用户操作

| 操作 | 说明 |
|------|------|
| 查看进度 | 查看任务状态和评论 |
| 确认完成 | 状态：`in_progress` → `completed` |
| 终止任务 | 状态：`in_progress` → `terminated` |

---

## 5. 时间记录

### 5.1 时间字段

| 字段 | 说明 | 填写时机 |
|------|------|---------|
| `started_at` | 开始时间 | 任务创建时自动填写 |
| `ended_at` | 结束时间 | 完成/终止时填写 |

### 5.2 任务时长

**计算公式**：
```
任务时长 = ended_at - started_at
```

**示例**：
```json
{
  "task_id": "task_001",
  "started_at": "2026-03-18T05:00:00Z",
  "ended_at": "2026-03-18T06:30:00Z",
  "duration": "1 小时 30 分钟"
}
```

## 6. 任务评论

### 6.1 评论功能

**支持的操作**：
- 添加评论
- 回复评论
- @提及 Agent
- 附加文件/链接

### 6.2 系统评论

**自动评论**：
- 任务创建
- 任务分配
- 任务完成
- 任务终止

**评论格式**：
```
[系统] 任务已分配给 Agent[程序员] (OpenClaw #1)
[系统] 任务已完成，耗时 1 小时 30 分钟
[系统] 任务已终止
```

## 7. 通知机制

### 7.1 通知类型

| 类型 | 触发场景 | 通知对象 |
|------|---------|---------|
| `task_assigned` | 任务创建/分配 | 执行者（Agent） |
| `task_completed` | 完成任务 | 用户 |
| `task_terminated` | 终止任务 | 执行者（Agent） |

### 7.2 通知方式

| 方式 | 说明 |
|------|------|
| WebSocket 推送 | 实时通知 |
| 站内通知 | 通知中心 |

## 8. API 接口

### 8.1 任务 CRUD

```
POST   /api/tasks                  # 创建任务（必须指定 Agent）
GET    /api/tasks                  # 获取任务列表
GET    /api/tasks/:id              # 获取任务详情
PUT    /api/tasks/:id              # 更新任务
DELETE /api/tasks/:id              # 删除任务
```

### 8.2 任务状态

```
POST   /api/tasks/:id/complete     # 完成任务
POST   /api/tasks/:id/terminate    # 终止任务
```

**完成任务**：
```json
POST /api/tasks/:id/complete
{
  "comment": "任务已完成"  // 可选
}

响应:
{
  "status": "completed",
  "ended_at": "2026-03-18T06:30:00Z"
}
```

**终止任务**：
```json
POST /api/tasks/:id/terminate
{
  "reason": "需求变更"  // 可选
}

响应:
{
  "status": "terminated",
  "ended_at": "2026-03-18T06:00:00Z"
}
```

### 8.3 任务分配

**创建任务时必须分配**：
```json
POST /api/tasks
{
  "title": "开发登录页面",
  "description": "...",
  "assignee": {
    "openclaw_id": "oc_001",
    "agent_id": "agent_A"
  }
}
```

## 9. 数据库设计

### 9.1 任务表

```sql
CREATE TABLE tasks (
    id              INTEGER PRIMARY KEY,
    title           TEXT NOT NULL,
    description     TEXT NOT NULL,
    status          TEXT DEFAULT 'in_progress',  -- in_progress, completed, terminated
    priority        TEXT DEFAULT 'medium',
    creator_id      INTEGER,
    assignee_openclaw_id  INTEGER REFERENCES openclaws(id) NOT NULL,
    assignee_agent_id     INTEGER REFERENCES agents(id) NOT NULL,
    started_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    ended_at        TIMESTAMP,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 9.2 任务评论表

```sql
CREATE TABLE task_comments (
    id              INTEGER PRIMARY KEY,
    task_id         INTEGER REFERENCES tasks(id),
    parent_id       INTEGER REFERENCES task_comments(id),
    author_id       INTEGER,
    author_type     TEXT,  -- 'user' or 'agent'
    content         TEXT NOT NULL,
    is_system       BOOLEAN DEFAULT FALSE,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
