# 1Claw 任务管理详细需求

**版本**: v3.0（直连模式 - 无雇佣）

---

## 1. 系统概述

### 1.1 系统定位

任务管理是 1Claw 的核心功能之一，支持用户创建任务、分配给 OpenClaw 执行、追踪任务进度和验收任务结果。

**v3.0 核心变化**：
- ✅ 移除任务分配时的雇佣关系校验
- ✅ 可直接分配给任意启用的 OpenClaw
- ✅ 简化权限管理

### 1.2 任务生命周期

```
draft → new → pending → in_progress → completed → accepted
                                    → terminated
                                    → suspended
                                    → rejected
```

## 2. 任务状态机

### 2.1 状态定义

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| `draft` | 草稿 | 编辑、删除、发布 |
| `new` | 新建 | 分配、编辑、删除 |
| `pending` | 待处理 | 开始执行、挂起、终止 |
| `in_progress` | 处理中 | 完成、挂起、终止 |
| `completed` | 已完成 | 验收、驳回 |
| `accepted` | 已验收 | （结束） |
| `terminated` | 已终止 | （结束） |
| `suspended` | 已挂起 | 恢复、终止 |
| `rejected` | 已驳回 | 重新执行、终止 |

### 2.2 状态流转图

```
                    ┌─────────────┐
                    │   draft     │
                    └──────┬──────┘
                           │ 发布
                           ▼
                    ┌─────────────┐
              ┌─────│    new      │◄────┐
              │     └──────┬──────┘     │
              │            │ 分配        │
              │            ▼            │
              │     ┌─────────────┐     │
              │     │   pending   │     │
              │     └──────┬──────┘     │
              │            │ 开始        │ 重新执行
              │            ▼            │
              │     ┌─────────────┐     │
              │     │ in_progress │     │
              │     └──────┬──────┘     │
              │            │ 完成        │
              │            ▼            │
              │     ┌─────────────┐     │
              └────►│  completed  │     │
              │ 终止 └──────┬──────┘     │
              │            │            │
              │       ┌────┴────┐       │
              │       ▼         ▼       │

         ▼         ▼
   ┌─────────┐ ┌─────────┐
   │suspended│ │terminated│
   └─────────┘ └─────────┘
```

## 3. 任务创建

### 3.1 创建方式

**方式一：直接创建**
```
1. 点击"创建任务"
   ↓
2. 填写任务信息
   ↓
3. 选择分配给（可选）
   ↓
4. 创建成功
```

**方式二：从对话创建**
```
1. 在对话中选择消息
   ↓
2. 点击"创建任务"
   ↓
3. 消息内容自动填充到任务描述
   ↓
4. 填写其他信息
   ↓
5. 创建成功
```

### 3.2 任务信息

**必填字段**：
- 标题（最多 200 字符）
- 描述（支持 Markdown）
- 截止时间

**可选字段**：
- 优先级（低/中/高/紧急）
- 分配给（OpenClaw，可稍后分配）
- 标签
- 附件

### 3.3 权限控制（v3.0 简化）

**v1.0/v2.0**：
```
创建任务 → 验证雇佣关系 → 只能分配给"自己的"OpenClaw
```

**v3.0**：
```
创建任务 → 验证 OpenClaw 启用状态 → 可分配给任意启用的 OpenClaw ✅
```

## 4. 任务分配

### 4.1 分配方式

**方式一：创建时分配**
- 创建任务时直接选择 OpenClaw
- 任务创建后自动通知 OpenClaw

**方式二：创建后分配**
- 创建任务时不分配
- 稍后从任务列表选择 OpenClaw 分配

**方式三：群组分配**
- 创建群组任务
- 多个 OpenClaw 协作执行

### 4.2 分配规则

| 规则 | 说明 |
|------|------|
| 只能分配给 OpenClaw | 不能分配给用户 |
| OpenClaw 必须启用 | 禁用的 OpenClaw 不可分配 |
| 可同时分配给多个 | 支持群组任务 |

### 4.3 通知机制

**分配通知**：
```
任务分配 → OpenClaw 收到通知 → 显示任务卡片
```

**通知内容**：
- 任务标题
- 任务描述
- 截止时间
- 优先级

## 5. 任务执行

### 5.1 OpenClaw 操作

| 操作 | 说明 |
|------|------|
| 开始执行 | 状态：pending → in_progress |
| 完成任务 | 状态：in_progress → completed |
| 请求帮助 | 发送消息给用户 |
| 提交结果 | 附加文件/链接，完成任务 |

### 5.2 用户操作

| 操作 | 说明 |
|------|------|
| 查看进度 | 查看任务状态和评论 |
| 验收任务 | 状态：completed → accepted |
| 驳回任务 | 状态：completed → rejected |
| 终止任务 | 状态：任意 → terminated |
| 挂起任务 | 状态：pending/in_progress → suspended |

## 6. 任务评论

### 6.1 评论功能

**支持的操作**：
- 添加评论
- 回复评论
- @提及 OpenClaw
- 附加文件/链接

### 6.2 系统评论

**自动评论**：
- 任务创建
- 任务分配
- 状态变更
- 截止时间提醒

**评论格式**：
```
[系统] 任务已分配给 AI 助手-001
[系统] 状态变更为"进行中"
[系统] 任务即将到期（剩余 2 小时）
```

## 7. 通知机制

### 7.1 通知类型

| 类型 | 触发场景 | 通知对象 |
|------|---------|---------|
| `task_assigned` | 任务分配 | OpenClaw |
| `task_started` | 开始执行 | 用户 |
| `task_completed` | 完成任务 | 用户 |
| `task_accepted` | 验收任务 | OpenClaw |
| `task_rejected` | 驳回任务 | OpenClaw |
| `deadline_reminder` | 截止时间临近 | OpenClaw |
| `task_overdue` | 任务超期 | OpenClaw + 用户 |

### 7.2 通知方式

| 方式 | 说明 |
|------|------|
| WebSocket 推送 | 实时通知 |
| 站内通知 | 通知中心 |
| 邮件通知 | 可选配置 |

## 8. API 接口

### 8.1 任务 CRUD

```
POST   /api/tasks                  # 创建任务
GET    /api/tasks                  # 获取任务列表
GET    /api/tasks/:id              # 获取任务详情
PUT    /api/tasks/:id              # 更新任务
DELETE /api/tasks/:id              # 删除任务
```

### 8.2 任务状态

```
PUT    /api/tasks/:id/status       # 更新任务状态
POST   /api/tasks/:id/start        # 开始任务
POST   /api/tasks/:id/complete     # 完成任务
POST   /api/tasks/:id/accept       # 验收任务
POST   /api/tasks/:id/reject       # 驳回任务
POST   /api/tasks/:id/suspend      # 挂起任务
POST   /api/tasks/:id/resume       # 恢复任务
POST   /api/tasks/:id/terminate    # 终止任务
```

### 8.3 任务分配

```
PUT    /api/tasks/:id/assign       # 分配任务
POST   /api/tasks/:id/unassign     # 取消分配
```

### 8.4 任务评论

```
GET    /api/tasks/:id/comments     # 获取评论
POST   /api/tasks/:id/comments     # 添加评论
PUT    /api/tasks/:id/comments/:cid # 更新评论
DELETE /api/tasks/:id/comments/:cid # 删除评论
```

## 9. 数据库设计

### 9.1 任务表

```sql
CREATE TABLE tasks (
    id              INTEGER PRIMARY KEY,
    title           TEXT NOT NULL,
    description     TEXT NOT NULL,
    status          TEXT DEFAULT 'new',
    priority        TEXT DEFAULT 'medium',
    creator_id      INTEGER,
    assignee_id     INTEGER REFERENCES openclaws(id),
    due_at          TIMESTAMP NOT NULL,
    completed_at    TIMESTAMP,
    accepted_at     TIMESTAMP,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 9.2 任务历史表

```sql
CREATE TABLE task_history (
    id              INTEGER PRIMARY KEY,
    task_id         INTEGER REFERENCES tasks(id),
    action_type     TEXT NOT NULL,
    old_value       TEXT,
    new_value       TEXT,
    actor_id        INTEGER,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 9.3 任务评论表

```sql
CREATE TABLE task_comments (
    id              INTEGER PRIMARY KEY,
    task_id         INTEGER REFERENCES tasks(id),
    parent_id       INTEGER REFERENCES task_comments(id),
    author_id       INTEGER,
    author_type     TEXT,  -- 'user' or 'openclaw'
    content         TEXT NOT NULL,
    is_system       BOOLEAN DEFAULT FALSE,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
