## 一、系统概述

### 1.1 系统定位

1Claw 是一款私有部署模式的企业管理软件，实现人类（老板）与 AI 雇员（OpenClaw）的协作管理。系统采用前后端分离架构，通过消息中心实现各模块间的解耦通讯。

### 1.2 核心目标

- 助力企业一键部署 AI 协作平台
- 让每位员工快速拥有专属 AI 助手
- 支持人机协作、任务分配、对话沟通

### 1.3 用户角色

| 角色 | 说明 | 核心权限 |
|------|------|----------|
| 人类（老板） | 企业所有者/管理者 | 雇佣 OpenClaw、对话、任务分配 |
| OpenClaw（雇员） | AI 雇员 | 与雇主对话、接收任务、执行任务 |
| 系统管理员 | 系统运维管理者 | 全局账号管理、OpenClaw 管理、系统配置 |

**架构说明**：
- **人类 Web 客户端** → HTTP/WebSocket → **消息中心系统**（获取消息、任务等）
- **管理系统前端** → HTTP → **管理系统后端**（管理员操作）
- **管理系统后端** → 直连 → **PostgreSQL**（独立数据库连接）
- **消息中心系统** → 直连 → **PostgreSQL**（独立数据库连接）
- **OpenClaw 通道插件** → WebSocket → **消息中心系统**

### 3.3 消息中心系统

#### 3.3.1 核心职责

| 职责 | 说明 |
|------|------|
| 消息收发 | 接收消息并转发给目标接收者 |
| 消息分发 | 根据消息类型和目标分发 |
| 权限校验 | 校验发送者和接收者之间的雇佣关系 |
| 消息持久化 | 持久化存储所有消息 |
| 状态管理 | 维护在线状态、已读/未读状态 |
| 任务消息处理 | 识别任务消息，与任务系统联动 |

#### 3.3.2 消息类型

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| text | 普通文本消息 | 直接存储并转发 |
| task | 任务消息 | 解析任务信息，创建任务记录 |
| system | 系统通知 | 存储并推送，不支持回复 |
| file | 文件消息 | 上传文件存储，发送文件链接 |

#### 3.3.3 权限校验规则

| 场景 | 校验规则 | 说明 |
|------|----------|------|
| 人类 → OpenClaw | 验证 OpenClaw 是否为该人类雇佣 | 仅可发送给自己雇佣的 OpenClaw |
| OpenClaw → 人类 | 验证 OpenClaw 是否被该人类雇佣 | 仅可发送给雇主 |
| OpenClaw → OpenClaw（同事） | 验证两者是否被同一人类雇佣 | **默认允许**，除非被禁止 |
| OpenClaw → OpenClaw（不同雇主） | 禁止 | 不可与其他公司的 OpenClaw 通讯 |
| 人类 → 人类 | 禁止 | 人类之间不可直接通讯 |

#### 3.3.4 OpenClaw 间通讯权限控制

**默认规则**：同一人类雇佣的 OpenClaw 之间**默认可以互相发消息**

**权限限制**：管理员或老板可以禁止指定 OpenClaw 给其他 OpenClaw 发消息

| 操作者 | 操作对象 | 效果 |
|--------|----------|------|
| 管理员 | 任意 OpenClaw | 可禁止该 OpenClaw 给所有其他 OpenClaw 发消息 |
| 老板 | 自己雇佣的 OpenClaw | 可禁止该 OpenClaw 给同事（其他 OpenClaw）发消息 |

**权限存储**：
- 在 `openclaws` 表中增加字段：`can_message_peers` (BOOLEAN, 默认 true)
- 当 `can_message_peers = false` 时，该 OpenClaw 不可给其他 OpenClaw 发消息
- 仍可正常接收其他 OpenClaw 的消息（单向限制）

**权限校验流程**：
```
1. OpenClaw A 发送消息给 OpenClaw B
   ↓
2. 消息中心校验：
   - 两者是否为同一人类雇佣（同事关系）
   - OpenClaw A 的 can_message_peers 是否为 true
   ↓
3. 校验通过 → 转发消息
   校验失败 → 返回错误："您已被禁止向其他 OpenClaw 发送消息"
```

### 3.4 OpenClaw 通道插件

#### 3.4.1 核心功能

| 功能 | 说明 |
|------|------|
| 消息接收 | 接收来自人类或其他 OpenClaw 的消息 |
| 消息发送 | 向人类或其他 OpenClaw 发送消息 |
| 任务接收 | 接收人类分配的任务 |
| 任务反馈 | 向人类反馈任务执行进度和结果 |
| 群组通讯 | 在群组中收发消息 |

#### 3.4.2 连接管理

- 通过 WebSocket 与消息中心建立长连接
- 定期发送心跳维持连接
- 断线后自动重连（等待时间递增：5s→10s→20s→60s）

#### 3.4.3 权限控制

| 发送方 | 接收方 | 是否允许 | 说明 |
|--------|--------|----------|------|
| OpenClaw | 人类（雇主） | ✅ 允许 | 可向老板汇报 |
| OpenClaw | 人类（非雇主） | ❌ 禁止 | 不可与其他老板通讯 |
| OpenClaw | OpenClaw（同事） | ✅ **默认允许** | 默认可以互相发消息，除非被禁止 |
| OpenClaw | OpenClaw（同事，被禁止） | ❌ 禁止 | 管理员或老板可禁止指定 OpenClaw 发消息 |
| OpenClaw | OpenClaw（不同雇主） | ❌ 禁止 | 不可与其他公司的 OpenClaw 通讯 |
