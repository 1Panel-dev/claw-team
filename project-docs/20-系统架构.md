## 一、系统概述

### 1.1 系统定位

1Claw 是一款私有部署模式的个人软件，所有 OpenClaw 直接连接到调度中心，为个人用户提供服务。无需雇佣流程，开箱即用。

### 1.2 核心目标

- 助力个人用户快速部署 AI 协作平台
- 所有 OpenClaw 直接可用，无需雇佣
- 支持人机协作、任务分配、对话沟通

### 1.3 核心对象

| 对象 | 说明 |
|------|------|
| 个人用户 | 软件使用者，拥有所有 OpenClaw |
| OpenClaw | AI 助手，直接连接到调度中心 |

**架构说明**：
- **个人用户** → Web 界面 → **调度中心系统**
- **所有 OpenClaw** → WebSocket → **调度中心系统**（直连，无需雇佣）
- **调度中心系统** → 本地文件 → **SQLite 数据库**

### 2.2 三大核心模块

| 模块 | 用户群体 | 架构形式 | 核心职责 |
|------|----------|----------|----------|
| 人类 Web 客户端 | 个人用户 | Web 应用 (Vue+Vite+TS) | OpenClaw 管理、任务分配、对话 |
| 调度中心系统 | 后端服务 | 纯后端 (Python+SQLite) | OpenClaw 连接、任务管理、消息路由 |
| OpenClaw 通道插件 | OpenClaw | OpenClaw Plugin | 消息收发、任务执行 |

### 2.3 技术栈总览

| 模块 | 前端 | 后端 | 数据库 | 通讯协议 |
|------|------|------|--------|----------|
| 人类 Web 客户端 | Vue + Vite + TypeScript | - | - | HTTP + WebSocket |
| 调度中心系统 | - | Python | SQLite | HTTP + WebSocket |
| OpenClaw 通道插件 | - | OpenClaw Plugin | - | WebSocket |

## 三、核心设计

### 3.1 直连模式（无雇佣）

**旧模式（v1.0/v2.0）**：
```
用户 → 雇佣 OpenClaw → 建立关系 → 才能使用
```

**新模式（v3.0）**：
```
用户 → 所有 OpenClaw 直接可用 ✅
```

**优势**：
- ✅ 无需雇佣流程
- ✅ 无需权限隔离
- ✅ 无需管理系统
- ✅ 开箱即用

### 3.2 OpenClaw 管理

**功能**：
| 功能 | 说明 |
|------|------|
| 添加 OpenClaw | 直接添加到调度中心 |
| 启用/禁用 | 控制 OpenClaw 可用性 |
| 配置角色 | 为 OpenClaw 配置角色和技能 |
| 查看状态 | 在线状态、任务执行情况 |

**状态**：
- `active` - 启用状态，可接受任务
- `disabled` - 禁用状态，不可用

### 3.3 任务管理

**任务分配**：
- 直接分配给指定 OpenClaw
- 或创建群组任务，多个 OpenClaw 协作

**任务状态**：
- `new` - 新建
- `in_progress` - 进行中
- `completed` - 已完成
- `accepted` - 已验收

### 3.4 消息通讯

**通讯模式**：
| 模式 | 说明 |
|------|------|
| 一对一 | 用户 ↔ OpenClaw |
| 群组 | 用户 + 多个 OpenClaw |
| OpenClaw 间 | OpenClaw ↔ OpenClaw（协作） |

## 四、数据库设计

### 4.1 核心表

| 表名 | 用途 |
|------|------|
| `openclaws` | OpenClaw 配置和状态 |
| `tasks` | 任务管理 |
| `messages` | 消息记录 |
| `roles` | 角色模板 |

### 4.2 关键设计

**OpenClaw 表**：
```sql
CREATE TABLE openclaws (
    id              INTEGER PRIMARY KEY,
    name            TEXT NOT NULL,
    description     TEXT,
    status          TEXT DEFAULT 'active',  -- active, disabled
    role_id         INTEGER REFERENCES roles(id),
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**无雇佣关系表**（已移除）：
- ~~`employments`~~ - 雇佣关系（不再需要）
- ~~`employment_history`~~ - 雇佣历史（不再需要）

## 五、API 设计

### 5.1 OpenClaw 管理 API

```
GET    /api/openclaws              # 获取所有 OpenClaw
POST   /api/openclaws              # 添加 OpenClaw
PUT    /api/openclaws/:id          # 更新 OpenClaw 配置
DELETE /api/openclaws/:id          # 删除 OpenClaw
POST   /api/openclaws/:id/enable   # 启用 OpenClaw
POST   /api/openclaws/:id/disable  # 禁用 OpenClaw
```

### 5.2 任务 API

```
POST   /api/tasks                  # 创建任务
GET    /api/tasks                  # 获取任务列表
GET    /api/tasks/:id              # 获取任务详情
PUT    /api/tasks/:id              # 更新任务
PUT    /api/tasks/:id/assign       # 分配任务给 OpenClaw
```

### 5.3 消息 API

```
POST   /api/messages/send          # 发送消息
GET    /api/messages/history       # 获取消息历史
GET    /api/messages/unread        # 获取未读消息
```
