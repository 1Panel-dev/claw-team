# 1Claw 系统架构设计文档

---

## 一、系统概述

### 1.1 系统定位

1Claw 是一款私有部署模式的个人软件，支持个人用户管理多个 OpenClaw，每个 OpenClaw 可承载多个 Agent（身份/员工），每个 Agent 有独立记忆和配置，实现多角色 AI 协作。

### 1.2 核心目标

- 助力个人用户快速部署 AI 协作平台
- 支持一个 OpenClaw 承载多个 Agent
- 每个 Agent 独立记忆、独立配置
- 支持跨 OpenClaw 的 Agent 协作

### 1.3 核心对象

| 对象 | 说明 |
|------|------|
| 个人用户 | 软件使用者，拥有所有 OpenClaw 和 Agent |
| OpenClaw | AI 连接实例，可承载多个 Agent |
| Agent（身份/员工） | AI 角色实例，拥有独立记忆和配置，执行具体任务 |

**架构说明**：
- **个人用户** → Web 界面 → **调度中心系统**
- **所有 OpenClaw** → WebSocket → **调度中心系统**（直连）
- **每个 OpenClaw** → 承载多个 Agent
- **每个 Agent** → 独立记忆、独立配置
- **调度中心系统** → 本地文件 → **SQLite 数据库**

### 2.2 核心模块

| 模块 | 用户群体 | 架构形式 | 核心职责 |
|------|----------|----------|----------|
| 人类 Web 客户端 | 个人用户 | Web 应用 (Vue+Vite+TS) | OpenClaw 管理、Agent 管理、任务分配、对话 |
| 调度中心系统 | 后端服务 | 纯后端 (Python+SQLite) | OpenClaw 连接、Agent 管理、任务管理、消息路由 |
| OpenClaw 通道插件 | OpenClaw | OpenClaw Plugin | 消息收发、多 Agent 管理、任务执行 |

### 2.3 技术栈总览

| 模块 | 前端 | 后端 | 数据库 | 通讯协议 |
|------|------|------|--------|----------|
| 人类 Web 客户端 | Vue + Vite + TypeScript | - | - | HTTP + WebSocket |
| 调度中心系统 | - | Python | SQLite | HTTP + WebSocket |
| OpenClaw 通道插件 | - | OpenClaw Plugin | - | WebSocket |

## 三、核心设计

### 3.1 层次结构

```
个人用户
│
├── OpenClaw #1
│   ├── Agent A [程序员] - 独立记忆
│   ├── Agent B [作家] - 独立记忆
│   └── Agent C [数据分析师] - 独立记忆
│
├── OpenClaw #2
│   ├── Agent D [翻译] - 独立记忆
│   └── Agent E [设计师] - 独立记忆
│
└── 群组对话
    ├── 用户 + Agent A + Agent D（跨 OpenClaw 协作）
    └── 用户 + Agent A + Agent B（同一 OpenClaw 内协作）
```

### 3.2 Agent（身份/员工）

**定义**：
- Agent 是 AI 的具体身份/角色实例
- 每个 Agent 有独立配置和记忆
- Agent 之间相互隔离
- 一个 OpenClaw 可承载多个 Agent
- 未配置 Agent 时使用 OpenClaw 默认 Agent

**Agent 信息**：
- Agent ID（唯一标识）
- Agent 名称
- 所属 OpenClaw ID
- 角色配置
- 技能列表
- 工作知识库
- 系统提示词
- 启用状态
- 独立记忆

### 3.3 对话模式

| 模式 | 说明 | 示例 |
|------|------|------|
| **一对一** | 用户 ↔ 某个 Agent | 用户 ↔ OpenClaw#1-AgentA[程序员] |
| **群组** | 用户 + 多个 Agent | 用户 + AgentA + AgentD（跨 OpenClaw） |

**群组规则**：
- 群组必须由用户创建
- 用户必须在群组中
- 群组可包含多个 Agent（可跨 OpenClaw）

### 3.4 任务分配

**分配规则**：
- 任务必须分配给具体的 Agent
- 不能分配给 OpenClaw 整体
- 可跨 OpenClaw 分配

**示例**：
```
任务：开发登录页面
分配给：
  - OpenClaw: oc_001
  - Agent: agent_A (程序员)

任务：设计 UI 界面
分配给：
  - OpenClaw: oc_002
  - Agent: agent_E (设计师)
```

## 四、数据库设计

### 4.1 核心表

| 表名 | 用途 |
|------|------|
| `openclaws` | OpenClaw 配置和状态 |
| `agents` | Agent 配置、状态、记忆引用 |
| `tasks` | 任务管理 |
| `messages` | 消息记录 |
| `groups` | 群组信息 |
| `group_members` | 群组成员（Agent） |
| `roles` | 角色模板 |

### 4.2 关键设计

**OpenClaw 表**：
```sql
CREATE TABLE openclaws (
    id              INTEGER PRIMARY KEY,
    name            TEXT NOT NULL,
    description     TEXT,
    status          TEXT DEFAULT 'active',
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**Agent 表**：
```sql
CREATE TABLE agents (
    id              INTEGER PRIMARY KEY,
    openclaw_id     INTEGER REFERENCES openclaws(id),
    name            TEXT NOT NULL,
    role            TEXT,
    config          JSON,
    memory_id       TEXT,  -- 独立记忆引用
    status          TEXT DEFAULT 'active',
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**任务表**：
```sql
CREATE TABLE tasks (
    id              INTEGER PRIMARY KEY,
    title           TEXT NOT NULL,
    description     TEXT NOT NULL,
    status          TEXT DEFAULT 'in_progress',  -- in_progress, completed, terminated
    priority        TEXT DEFAULT 'medium',
    creator_id      INTEGER,
    assignee_openclaw_id  INTEGER REFERENCES openclaws(id) NOT NULL,
    assignee_agent_id     INTEGER REFERENCES agents(id) NOT NULL,  -- 必须指定具体 Agent
    started_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,  -- 开始时间（=创建时间）
    ended_at        TIMESTAMP,  -- 结束时间（完成/终止时填写）
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 五、API 设计

### 5.1 OpenClaw 管理 API

```
GET    /api/openclaws              # 获取所有 OpenClaw
POST   /api/openclaws              # 添加 OpenClaw
PUT    /api/openclaws/:id          # 更新 OpenClaw 配置
DELETE /api/openclaws/:id          # 删除 OpenClaw
POST   /api/openclaws/:id/enable   # 启用 OpenClaw
POST   /api/openclaws/:id/disable  # 禁用 OpenClaw
```

### 5.2 Agent 管理 API

```
GET    /api/openclaws/:id/agents        # 获取 OpenClaw 下的所有 Agent
POST   /api/openclaws/:id/agents        # 创建 Agent
GET    /api/agents/:id                  # 获取 Agent 详情
PUT    /api/agents/:id                  # 更新 Agent 配置
DELETE /api/agents/:id                  # 删除 Agent
POST   /api/agents/:id/enable           # 启用 Agent
POST   /api/agents/:id/disable          # 禁用 Agent
GET    /api/agents/:id/memory           # 获取 Agent 记忆
```

### 5.3 任务 API

```
POST   /api/tasks                  # 创建任务（必须指定 Agent）
GET    /api/tasks                  # 获取任务列表
GET    /api/tasks/:id              # 获取任务详情
PUT    /api/tasks/:id              # 更新任务
POST   /api/tasks/:id/complete     # 完成任务
POST   /api/tasks/:id/terminate    # 终止任务
```

### 5.4 群组 API

```
POST   /api/groups                 # 创建群组
GET    /api/groups                 # 获取群组列表
GET    /api/groups/:id             # 获取群组详情
PUT    /api/groups/:id/members     # 更新群组成员
DELETE /api/groups/:id             # 删除群组
```
