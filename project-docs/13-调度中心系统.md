# 1Claw 调度中心系统需求分析

**文档版本**: v3.0  
**创建时间**: 2026-03-12  
**最后更新**: 2026-03-18  
**定位**: 个人软件（直连模式 - 无雇佣）

---

## 1. 系统概述

### 1.1 系统定位

调度中心系统是 1Claw 的后端服务核心，作为 OpenClaw 连接中枢，为人类 Web 客户端、OpenClaw 通道插件提供消息收发、任务管理、通知服务。

**核心变化（v3.0）**：
- ✅ 移除雇佣关系校验
- ✅ 所有 OpenClaw 直接连接
- ✅ 简化权限管理

### 1.2 核心职责

| 职责 | 说明 |
|------|------|
| OpenClaw 管理 | 所有 OpenClaw 直接注册、启用/禁用 |
| 消息收发 | 接收消息并转发给目标接收者 |
| 消息分发 | 根据消息类型和目标分发 |
| 任务管理 | 任务创建、分配、状态追踪、截止时间提醒 |
| 消息持久化 | 持久化存储所有消息 |
| 状态管理 | 维护在线状态、已读/未读状态、任务状态 |
| 通知服务 | 任务超期提醒、状态变更通知 |
| 定时任务 | 截止时间提醒、心跳检查、数据清理 |

### 1.3 用户群体

| 用户 | 说明 |
|------|------|
| 人类 Web 客户端 | 通过调度中心管理 OpenClaw、分配任务、通讯 |
| OpenClaw 通道插件 | 通过调度中心接收和发送消息、接收任务 |

---

## 2. 系统关系

### 2.1 与客户端的关系

**系统关系**：
```
个人用户 → Web 客户端 → 调度中心 ←→ 所有 OpenClaw（直连）
```

**关系说明**：
- 所有 OpenClaw 直接连接到调度中心
- 调度中心负责消息中转、任务管理
- **无需雇佣关系校验**（v3.0 简化）

---

## 3. 功能模块

调度中心系统包含以下功能模块：

| 模块 | 说明 |
|------|------|
| OpenClaw 管理 | OpenClaw 注册、启用/禁用、配置 |
| 消息通讯 | 消息收发、路由、持久化 |
| 任务管理 | 任务全生命周期管理 |
| 通知服务 | 任务提醒、状态变更通知 |
| 文件服务 | 工作空间文件上传下载 |

## 4. OpenClaw 管理

### 4.1 OpenClaw 注册

**注册流程**：
```
1. OpenClaw 启动
   ↓
2. 连接到调度中心 WebSocket
   ↓
3. 发送注册请求（包含 OpenClaw ID、配置）
   ↓
4. 调度中心验证并记录
   ↓
5. 返回注册成功
   ↓
6. OpenClaw 进入可用状态
```

**注册信息**：
- OpenClaw ID
- OpenClaw 名称
- 角色配置
- 技能列表
- 在线状态

### 4.2 OpenClaw 状态

| 状态 | 说明 | 可接受任务 |
|------|------|-----------|
| `active` | 启用状态 | ✅ |
| `disabled` | 禁用状态 | ❌ |
| `offline` | 离线状态 | ❌ |

### 4.3 OpenClaw 配置

**可配置项**：
- 名称
- 描述
- 角色
- 技能
- 工作知识库
- 启用/禁用状态

## 5. 消息通讯服务

### 5.1 身份验证

**验证内容**：
- OpenClaw ID 是否存在
- OpenClaw 是否启用
- 用户身份（可选，单用户模式可简化）

### 5.2 连接管理

**连接状态**：
- 连接中：正在建立连接
- 已连接：连接正常
- 断开：连接断开（网络故障、服务端关闭）

**心跳机制**：
- 客户端定期发送心跳
- 服务端响应心跳
- 超时未响应则判定离线

### 5.3 消息路由

#### 消息类型

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| `text` | 普通文本消息 | 直接存储并转发 |
| `task` | 任务消息 | 解析任务信息，创建/更新任务记录 |
| `system` | 系统通知 | 存储并推送，不支持回复 |
| `file` | 文件消息 | 上传文件存储，发送文件链接 |

#### 路由规则

**消息路由流程**：
1. 接收消息
2. 验证发送者身份
3. 解析消息类型
4. 根据类型处理
5. 查询接收者在线状态
6. 如在线则推送，如离线则存入离线消息队列
7. 返回发送结果

**v3.0 简化**：
- ❌ 移除雇佣关系校验
- ✅ 所有启用的 OpenClaw 可直接通讯

### 5.4 离线消息处理

**离线消息存储**：
- 用户离线时，消息存储到离线队列
- 设置过期时间（如 7 天）
- 上线后批量推送

## 6. 任务管理服务

### 6.1 任务生命周期

**9 种任务状态**：
- `draft` - 草稿
- `new` - 新建
- `pending` - 待处理
- `in_progress` - 处理中
- `completed` - 已完成
- `accepted` - 已验收
- `terminated` - 已终止
- `suspended` - 已挂起
- `rejected` - 已驳回

### 6.2 核心功能

- **任务创建**：用户和 OpenClaw 都可创建任务
- **任务分配**：分配给指定 OpenClaw
- **截止时间**：必填字段，超期自动提醒
- **任务评论**：状态变更时可填写评论
- **通知机制**：覆盖任务全流程

### 6.3 API 接口

```
POST   /api/tasks                 # 创建任务
GET    /api/tasks                 # 获取任务列表
GET    /api/tasks/:id             # 获取任务详情
PUT    /api/tasks/:id             # 更新任务
PUT    /api/tasks/:id/status      # 更新任务状态
PUT    /api/tasks/:id/assign      # 分配任务
GET    /api/tasks/:id/comments    # 获取任务评论
POST   /api/tasks/:id/comments    # 添加评论
```

## 7. 通知服务

### 7.1 通知类型

| 类型 | 说明 |
|------|------|
| `task_assigned` | 任务分配 |
| `task_completed` | 任务完成 |
| `task_accepted` | 任务验收 |
| `task_rejected` | 任务驳回 |
| `deadline_reminder` | 截止时间提醒 |
| `status_changed` | 状态变更 |

### 7.2 通知触发场景

| 场景 | 通知对象 |
|------|---------|
| 任务分配 | 执行者（OpenClaw） |
| 任务完成 | 创建者（用户） |
| 任务验收 | 执行者（OpenClaw） |
| 截止时间临近 | 执行者（OpenClaw） |
| 任务超期 | 执行者（OpenClaw）+ 创建者（用户） |

## 8. 文件传输服务

### 8.1 文件上传

**上传流程**：
1. 客户端请求上传文件
2. 验证权限
3. 检查工作空间配额
4. 接收文件流
5. 存储文件到工作空间目录
6. 返回文件信息

### 8.2 文件存储

**存储结构**：
```
/storage/
├── workspace/
│   ├── oc_001/          # OpenClaw 001 的工作空间
│   ├── oc_002/          # OpenClaw 002 的工作空间
│   └── ...
└── temp/                # 临时文件
```

**配额管理**：
- 每个 OpenClaw 默认 1GB
- 可调整配额
- 定期清理过期临时文件

## 9. 定时任务

### 9.1 定时任务类型

| 任务 | 触发频率 | 说明 |
|------|---------|------|
| 截止时间提醒 | 每 10 分钟 | 检查即将到期的任务 |
| 心跳检查 | 每 30 秒 | 检查 OpenClaw 在线状态 |
| 数据清理 | 每天凌晨 | 清理过期临时文件、日志 |
| 状态扫描 | 每小时 | 扫描逾期任务并标记 |

### 9.2 实现方式

**工具**：APScheduler

**配置示例**：
```python
from apscheduler.schedulers.asyncio import AsyncIOScheduler

scheduler = AsyncIOScheduler()

# 添加定时任务
scheduler.add_job(
    check_task_deadlines,
    trigger='interval',
    minutes=10,
    id='task_deadline_check'
)

scheduler.start()
```
