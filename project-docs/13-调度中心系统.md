# 1Claw 调度中心系统需求分析

---

## 1. 系统概述

### 1.1 系统定位

调度中心系统是 1Claw 的后端服务核心，作为 OpenClaw 和 Agent 连接中枢，为人类 Web 客户端、OpenClaw 通道插件提供消息收发、Agent 管理、任务管理、通知服务。

### 1.2 核心职责

| 职责 | 说明 |
|------|------|
| OpenClaw 管理 | OpenClaw 注册、启用/禁用 |
| Agent 管理 | Agent 创建、配置、独立记忆管理 |
| 消息收发 | 接收消息并转发给目标接收者 |
| 消息分发 | 根据消息类型和目标分发 |
| 任务管理 | 任务创建、分配、状态追踪、截止时间提醒 |
| 消息持久化 | 持久化存储所有消息 |
| 状态管理 | 维护在线状态、已读/未读状态、任务状态 |
| 通知服务 | 任务超期提醒、状态变更通知 |
| 定时任务 | 截止时间提醒、心跳检查、数据清理 |

### 1.3 用户群体

| 用户 | 说明 |
|------|------|
| 人类 Web 客户端 | 通过调度中心管理 OpenClaw、Agent、分配任务、通讯 |
| OpenClaw 通道插件 | 通过调度中心接收和发送消息、接收任务、管理多个 Agent |

---

## 2. 系统关系

### 2.1 与客户端的关系

**系统关系**：
```
个人用户 → Web 客户端 → 调度中心 ←→ OpenClaw #1 ← Agent A, B, C
                                      ↘ OpenClaw #2 ← Agent D, E
```

**关系说明**：
- 所有 OpenClaw 直接连接到调度中心
- 每个 OpenClaw 可承载多个 Agent
- 每个 Agent 有独立记忆和配置
- 调度中心负责消息中转、任务管理

---

## 3. 功能模块

调度中心系统包含以下功能模块：

| 模块 | 说明 |
|------|------|
| OpenClaw 管理 | OpenClaw 注册、启用/禁用、配置 |
| Agent 管理 | Agent 创建、配置、独立记忆管理 |
| 消息通讯 | 消息收发、路由、持久化 |
| 任务管理 | 任务全生命周期管理 |
| 通知服务 | 任务提醒、状态变更通知 |
| 文件服务 | 工作空间文件上传下载 |

## 4. OpenClaw 管理

### 4.1 OpenClaw 注册

**注册流程**：
```
1. OpenClaw 启动
   ↓
2. 连接到调度中心 WebSocket
   ↓
3. 发送注册请求（包含 OpenClaw ID、配置）
   ↓
4. 调度中心验证并记录
   ↓
5. 返回注册成功
   ↓
6. OpenClaw 进入可用状态
```

**注册信息**：
- OpenClaw ID
- OpenClaw 名称
- 默认配置

### 4.2 OpenClaw 状态

| 状态 | 说明 | 可承载 Agent |
|------|------|-------------|
| `active` | 启用状态 | ✅ |
| `disabled` | 禁用状态 | ❌ |
| `offline` | 离线状态 | ❌ |

---

## 5. Agent（身份/员工）管理

### 5.1 Agent 定义

**Agent**：
- AI 的具体身份/角色实例
- 每个 Agent 有独立配置和记忆
- Agent 之间相互隔离
- 一个 OpenClaw 可承载多个 Agent
- 未配置 Agent 时使用 OpenClaw 默认 Agent

### 5.2 Agent 创建

**创建方式**：

| 方式 | 说明 |
|------|------|
| **自己创建** | 用户自定义 Agent 的名称、角色、技能、提示词等 |
| **使用预设** | 选择系统预设的 Agent 模板（如：程序员、作家、设计师等） |

**自己创建流程**：
```
1. 用户通过 Web 界面创建 Agent
   ↓
2. 填写 Agent 信息
   - 名称
   - 角色
   - 技能配置
   - 系统提示词
   ↓
3. 调度中心保存 Agent 配置
   ↓
4. 通知对应 OpenClaw
   ↓
5. Agent 创建成功 ✅
```

**使用预设流程**：
```
1. 用户选择预设 Agent 模板
   ↓
2. 选择要添加到的 OpenClaw
   ↓
3. 可选：修改预设配置（名称、提示词等）
   ↓
4. 调度中心保存 Agent 配置
   ↓
5. Agent 创建成功 ✅
```

**预设 Agent 模板**：
- 程序员
- 作家
- 数据分析师
- 设计师
- 翻译
- 产品经理
- 更多预设模板可扩展

**Agent 信息**：
- Agent ID（唯一标识）
- Agent 名称
- 所属 OpenClaw ID
- 角色配置
- 技能列表
- 工作知识库
- 系统提示词
- 启用状态
- 来源（自定义/预设）

### 5.3 Agent 独立记忆

**记忆存储**：
- 每个 Agent 有独立的记忆存储
- 记忆与 Agent ID 绑定
- Agent 之间记忆不共享

**记忆内容**：
- 对话历史
- 任务执行记录
- 用户偏好
- 工作知识

**记忆 API**：
```
GET    /api/agents/:id/memory        # 获取 Agent 记忆
POST   /api/agents/:id/memory        # 添加记忆
DELETE /api/agents/:id/memory        # 清除记忆
```

### 5.4 Agent 状态

| 状态 | 说明 | 可接受任务 |
|------|------|-----------|
| `active` | 启用状态 | ✅ |
| `disabled` | 禁用状态 | ❌ |

## 6. 消息通讯服务

### 6.1 身份验证

**验证内容**：
- OpenClaw ID 是否存在
- OpenClaw 是否启用
- Agent ID 是否存在（如果指定）
- Agent 是否启用（如果指定）

### 6.2 连接管理

**连接状态**：
- 连接中：正在建立连接
- 已连接：连接正常
- 断开：连接断开（网络故障、服务端关闭）

**心跳机制**：
- 客户端定期发送心跳
- 服务端响应心跳
- 超时未响应则判定离线

### 6.3 消息路由

#### 消息类型

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| `text` | 普通文本消息 | 直接存储并转发 |
| `task` | 任务消息 | 解析任务信息，创建/更新任务记录 |
| `system` | 系统通知 | 存储并推送，不支持回复 |
| `file` | 文件消息 | 上传文件存储，发送文件链接 |

#### 路由规则

**消息路由流程**：
1. 接收消息
2. 验证发送者身份
3. 解析消息类型
4. 根据类型处理
5. 查询接收者在线状态
6. 如在线则推送，如离线则存入离线消息队列
7. 返回发送结果

**消息寻址**：
```
# 一对一消息
{
  "from": "user",
  "to": {
    "openclaw_id": "oc_001",
    "agent_id": "agent_A"  # 指定具体 Agent
  }
}

# 群组消息
{
  "from": "user",
  "to": {
    "group_id": "group_001"
  }
}
```

### 6.4 群组管理

**群组规则**：
- 群组必须由用户创建
- 用户必须在群组中
- 群组成员为多个 Agent（可跨 OpenClaw）

**群组信息**：
- 群组 ID
- 群组名称
- 创建者（用户）
- 成员列表（Agent ID 列表）
- 创建时间

### 6.5 离线消息处理

**离线消息存储**：
- 用户离线时，消息存储到离线队列
- 设置过期时间（如 7 天）
- 上线后批量推送

## 7. 任务管理服务

### 7.1 任务生命周期

**3 种任务状态**：
- `in_progress` - 正在执行（创建后默认状态）
- `completed` - 已完成
- `terminated` - 已终止

### 7.2 任务分配

**分配规则**：
- 任务创建时必须分配给具体的 Agent
- 不能分配给 OpenClaw 整体
- 可跨 OpenClaw 分配

**任务信息**：
```json
{
  "task_id": "task_001",
  "title": "开发登录页面",
  "description": "...",
  "assignee": {
    "openclaw_id": "oc_001",
    "agent_id": "agent_A"  # 必须指定具体 Agent
  },
  "status": "in_progress",
  "started_at": "2026-03-18T05:00:00Z",
  "ended_at": null
}
```

### 7.3 核心功能

- **任务创建**：用户和 Agent 都可创建任务，创建时必须指定执行者
- **任务分配**：创建时分配给指定 Agent
- **任务评论**：状态变更时可填写评论
- **通知机制**：任务创建、完成、终止通知
- **时间记录**：自动记录开始时间（创建时间）和结束时间（完成/终止时间）

## 8. 通知服务

### 8.1 通知类型

| 类型 | 说明 |
|------|------|
| `task_assigned` | 任务分配 |
| `task_started` | 开始执行 |
| `task_completed` | 完成任务 |
| `task_accepted` | 验收任务 |
| `task_rejected` | 驳回任务 |
| `deadline_reminder` | 截止时间提醒 |
| `status_changed` | 状态变更 |

### 8.2 通知触发场景

| 场景 | 通知对象 |
|------|---------|
| 任务分配 | 执行者（Agent） |
| 任务完成 | 创建者（用户） |
| 任务验收 | 执行者（Agent） |
| 截止时间临近 | 执行者（Agent） |
| 任务超期 | 执行者（Agent）+ 创建者（用户） |

---

## 9. 定时任务

### 9.1 定时任务类型

| 任务 | 触发频率 | 说明 |
|------|---------|------|
| 心跳检查 | 每 30 秒 | 检查 OpenClaw 在线状态 |
| 数据清理 | 每天凌晨 | 清理过期临时文件、日志 |

### 9.2 实现方式

**工具**：APScheduler

## 10. 数据库设计

### 10.1 核心表

| 表名 | 用途 |
|------|------|
| `openclaws` | OpenClaw 配置和状态 |
| `agents` | Agent 配置、状态、记忆引用 |
| `tasks` | 任务管理 |
| `task_history` | 任务历史记录 |
| `task_comments` | 任务评论 |
| `messages` | 消息记录 |
| `groups` | 群组信息 |
| `group_members` | 群组成员（Agent） |
| `notifications` | 通知记录 |
| `roles` | 角色模板 |

### 10.2 Agent 表设计

```sql
CREATE TABLE agents (
    id              INTEGER PRIMARY KEY,
    openclaw_id     INTEGER REFERENCES openclaws(id),
    name            TEXT NOT NULL,
    role            TEXT,
    config          JSON,
    memory_id       TEXT,  -- 独立记忆引用
    status          TEXT DEFAULT 'active',
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
