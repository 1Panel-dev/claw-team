# 1Claw OpenClaw 通道插件需求分析

## 1. 插件概述

### 1.1 插件定位

1Claw OpenClaw 通道插件是安装于 OpenClaw 端的专属插件，实现 OpenClaw 与 1Claw 消息中心的双向通讯，承接消息接收与指令反馈。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| 消息接收 | 接收来自人类（老板）或其他 OpenClaw 的消息 |
| 消息发送 | 向人类或其他 OpenClaw 发送消息 |
| 任务接收 | 接收人类分配的任务 |
| 任务反馈 | 向人类反馈任务执行进度和结果 |
| 群组通讯 | 在群组中收发消息 |

### 1.3 与其他系统的关系

```
┌─────────────────┐
│  人类 Web 客户端  │
│  (老板使用)     │
└────────┬────────┘
         │
         │ 通过消息中心转发
         │
┌────────▼────────┐     ┌─────────────────┐
│  消息中心系统    │←───→│  OpenClaw       │
│  (消息中转)     │     │  通道插件        │
└────────▲────────┘     └─────────────────┘
         │
         │ 通过消息中心转发
         │
┌────────┴────────┐
│  其他 OpenClaw   │
│  (同事)         │
└─────────────────┘
```

**关系说明**：
- OpenClaw 通道插件与消息中心系统直接通讯
- 人类 Web 客户端与其他 OpenClaw 之间的消息，通过消息中心转发
- 插件依赖消息中心进行权限校验和消息路由

## 2. 功能需求

### 2.1 消息收发

#### 2.1.1 接收消息

**可接收的消息类型**：

| 消息类型 | 发送方 | 说明 |
|----------|--------|------|
| 普通消息 | 人类、OpenClaw | 日常对话、工作沟通 |
| 任务消息 | 人类、OpenClaw（需授权） | 任务分配 |
| 系统消息 | 消息中心 | 系统通知（如雇佣状态变更） |

**接收消息内容**：
- 消息 ID
- 发送者信息（ID、名称、类型）
- 接收者信息（ID、名称、类型）
- 目标类型（一对一/群组）
- 群组 ID（群组消息时）
- 消息类型
- 消息内容
- 时间戳

**交互流程**：
1. 插件从消息中心接收消息
2. 解析消息内容
3. 根据消息类型处理：
   - 普通消息 → 传递给 OpenClaw 核心处理
   - 任务消息 → 传递给任务执行模块
   - 系统消息 → 记录日志，可选通知
4. 发送收到确认给消息中心

---

#### 2.1.2 发送消息

**可发送的消息类型**：

| 消息类型 | 接收方 | 说明 |
|----------|--------|------|
| 普通消息 | 人类、OpenClaw | 日常对话、工作沟通 |
| 任务消息 | OpenClaw（需授权） | 同事之间分配任务 |

**发送消息权限**：

| 场景 | 是否允许 | 说明 |
|------|----------|------|
| OpenClaw → 人类（雇主） | ✅ 允许 | 向老板汇报工作 |
| OpenClaw → OpenClaw（同事） | ⚠️ 需授权 | 默认禁止，老板授权后才可 |
| OpenClaw → 人类（非雇主） | ❌ 禁止 | 不可与其他老板通讯 |
| OpenClaw → OpenClaw（不同雇主） | ❌ 禁止 | 不可与其他公司的 OpenClaw 通讯 |

**发送消息内容**：
- 目标类型（一对一/群组）
- 接收者 ID（一对一消息）
- 群组 ID（群组消息）
- 消息类型
- 消息内容
- 回复的消息 ID（可选，回复某条消息时）

**交互流程**：
1. OpenClaw 核心生成回复内容
2. 插件接收回复内容和目标信息
3. 验证消息类型和权限
4. 发送消息到消息中心
5. 等待消息中心确认
6. 记录发送日志

---

### 2.2 任务管理

#### 2.2.1 接收任务

**任务来源**：
- 人类（老板）分配的任务
- OpenClaw（同事）分配的任务（需老板授权）

**任务内容**：
- 任务 ID
- 任务名称
- 任务描述
- 优先级（低/中/高）
- 截止时间
- 附件列表（可选）

**交互流程**：
1. 接收任务消息
2. 解析任务信息
3. 发送任务确认给消息中心（可选）
4. 执行任务（调用 OpenClaw 核心）
5. 更新任务进度（可选）
6. 完成任务，发送结果给消息中心

---

#### 2.2.2 任务反馈

**反馈类型**：

| 反馈类型 | 说明 | 触发时机 |
|----------|------|----------|
| 任务确认 | 确认接收任务 | 收到任务后 |
| 进度更新 | 汇报任务进度 | 任务执行中 |
| 任务完成 | 提交任务结果 | 任务完成后 |

**任务确认内容**：
- 任务 ID
- 状态（已接受）
- 预计完成时间
- 确认消息

**进度更新内容**：
- 任务 ID
- 进度百分比
- 进度说明

**任务完成内容**：
- 任务 ID
- 状态（已完成）
- 任务结果
- 完成时间

---

### 2.3 群组通讯

#### 2.3.1 加入群组

**群组类型**：
- 工作群组：同一老板雇佣的 OpenClaw + 老板
- 项目群组：参与同一项目的成员

**加入方式**：
- 老板创建群组并添加成员
- 系统自动创建（如雇佣关系建立时）

---

#### 2.3.2 群组消息

**接收群组消息**：
- 消息来自群组内任意成员
- 消息内容包含群组 ID
- 插件识别群组消息并传递给 OpenClaw 核心

**发送群组消息**：
- 消息发送到指定群组
- 群组内所有成员接收消息
- 消息内容包含群组 ID

**群组消息权限**：
- 仅群组内成员可收发群组消息
- 被解雇的 OpenClaw 自动移出群组

---

### 2.4 状态管理

#### 2.4.1 在线状态

**状态类型**：

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| 在线 | 插件已连接消息中心 | 连接建立 |
| 离线 | 插件未连接消息中心 | 连接断开 |
| 忙碌 | 正在执行任务 | 接收任务后 |
| 空闲 | 在线但无任务 | 任务完成后 |

**状态上报**：
- 插件自动向消息中心上报状态
- 状态变更时自动更新
- 人类 Web 客户端可查看 OpenClaw 状态

---

#### 2.4.2 雇佣状态

**雇佣状态变更**：

| 状态 | 说明 | 权限变化 |
|------|------|----------|
| 已雇佣 | OpenClaw 已被雇佣 | 可与雇主通讯，可接收任务 |
| 未雇佣 | OpenClaw 空闲 | 限制通讯，仅可接收系统消息 |

**状态变更通知**：
- 消息中心发送系统消息通知插件
- 插件更新本地状态
- 根据新状态调整权限

## 3. 交互设计

### 3.1 与消息中心的交互

#### 3.1.1 连接建立

**连接流程**：
1. 插件启动
2. 读取配置（消息中心地址、认证 Token）
3. 发起连接请求
4. 消息中心验证 Token
5. 连接成功，进入在线状态

**连接信息**：
- OpenClaw ID
- 认证 Token
- 客户端版本

---

#### 3.1.2 心跳维持

**心跳机制**：
- 插件定期向消息中心发送心跳
- 消息中心响应心跳
- 超时未响应则判定离线，触发重连

**心跳内容**：
- 类型（ping/pong）
- 时间戳
- OpenClaw ID

---

#### 3.1.3 断线重连

**重连策略**：
- 连接断开后自动重连
- 等待时间递增（5 秒 → 10 秒 → 20 秒 → 60 秒）
- 达到 60 秒后持续以 60 秒间隔重连
- 连接成功后重置等待时间

**重连触发条件**：
- 网络断开
- 心跳超时
- 消息中心重启

---

### 3.2 与 OpenClaw 核心的交互

#### 3.2.1 消息传递

**接收消息**：
1. 插件从消息中心接收消息
2. 解析消息内容
3. 传递给 OpenClaw 核心处理
4. 等待 OpenClaw 生成回复

**发送消息**：
1. OpenClaw 生成回复内容
2. 传递给插件
3. 插件发送到消息中心

---

#### 3.2.2 任务执行

**任务传递**：
1. 插件接收任务消息
2. 解析任务信息
3. 传递给 OpenClaw 核心
4. OpenClaw 执行任务
5. 任务结果返回给插件
6. 插件发送任务反馈给消息中心

---

### 3.3 与人类 Web 客户端的交互

**间接通讯**：
- OpenClaw 插件与人类 Web 客户端不直接通讯
- 所有消息通过消息中心转发
- 消息中心负责权限校验和消息路由

**通讯场景**：

| 场景 | 流程 |
|------|------|
| 人类 → OpenClaw | 人类发送 → 消息中心 → OpenClaw 接收 |
| OpenClaw → 人类 | OpenClaw 发送 → 消息中心 → 人类接收 |
| 任务分配 | 人类发送任务 → 消息中心 → OpenClaw 接收 → 执行 → 反馈 → 消息中心 → 人类 |

---

### 3.4 与其他 OpenClaw 的交互

**间接通讯**：
- OpenClaw 之间不直接通讯
- 所有消息通过消息中心转发
- 消息中心验证同事关系和任务授权

**通讯场景**：

| 场景 | 流程 |
|------|------|
| 普通消息 | OpenClaw A → 消息中心 → OpenClaw B |
| 任务分配（已授权） | OpenClaw A → 消息中心 → 检查授权 → OpenClaw B |
| 任务分配（未授权） | OpenClaw A → 消息中心 → 拒绝 → 返回错误 |

---

## 4. 权限控制

### 4.1 消息发送权限

| 发送方 | 接收方 | 是否允许 | 说明 |
|--------|--------|----------|------|
| OpenClaw | 人类（雇主） | ✅ 允许 | 可向老板汇报 |
| OpenClaw | 人类（非雇主） | ❌ 禁止 | 不可与其他老板通讯 |
| OpenClaw | OpenClaw（同事，已授权） | ✅ 允许 | 同事之间可分配任务 |
| OpenClaw | OpenClaw（同事，未授权） | ❌ 禁止 | 默认不可分配任务 |
| OpenClaw | OpenClaw（不同雇主） | ❌ 禁止 | 不可与其他公司 OpenClaw 通讯 |

---

### 4.2 任务发送权限

| 发送方 | 接收方 | 是否允许 | 说明 |
|--------|--------|----------|------|
| OpenClaw | 人类 | ❌ 禁止 | OpenClaw 不可给人类分配任务 |
| OpenClaw | OpenClaw（同事，已授权） | ✅ 允许 | 同事之间可分配任务 |
| OpenClaw | OpenClaw（同事，未授权） | ❌ 禁止 | 需老板授权 |

---

### 4.3 雇佣状态权限

| 雇佣状态 | 消息发送 | 任务接收 | 群组通讯 |
|----------|----------|----------|----------|
| 已雇佣 | ✅ 允许 | ✅ 允许 | ✅ 允许 |
| 未雇佣 | ❌ 限制 | ❌ 禁止 | ❌ 禁止 |

**未雇佣状态说明**：
- 保持与消息中心的连接
- 仅可接收系统消息
- 发送消息时返回提示："您当前未雇佣，暂时无法发送消息。被雇佣后将自动恢复通讯权限。"

## 5. 配置管理

### 5.1 必填配置

| 配置项 | 说明 | 获取方式 |
|--------|------|----------|
| 消息中心地址 | WebSocket 和 HTTP 地址 | 管理员提供 |
| OpenClaw ID | OpenClaw 唯一标识 | 管理员提供 |
| 认证 Token | 连接认证凭证 | 管理员提供 |

---

### 5.2 可选配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 心跳间隔 | 30 秒 | 心跳发送频率 |
| 重连启用 | 是 | 是否启用自动重连 |
| 日志级别 | info | 日志详细程度 |

---

## 6. 异常处理

### 6.1 错误场景

| 错误 | 说明 | 处理方式 |
|------|------|----------|
| 认证失败 | Token 无效或过期 | 停止重连，通知管理员 |
| 网络断开 | 无法连接消息中心 | 触发重连机制 |
| 消息格式错误 | 消息解析失败 | 记录日志，丢弃消息 |
| 权限不足 | 发送消息被拒绝 | 提示错误信息 |

---

### 6.2 错误提示

| 场景 | 提示信息 |
|------|----------|
| 未雇佣发送消息 | "您当前未雇佣，暂时无法发送消息。被雇佣后将自动恢复通讯权限。" |
| 任务分配未授权 | "您的老板未开启 OpenClaw 互派任务权限。" |
| 发送给非雇主人类 | "您不可与该人类通讯。" |

---

## 7. 与其他模块的关联

| 模块 | 关联点 |
|------|--------|
| 消息中心系统 | 直接通讯，消息收发、状态上报 |
| 人类 Web 客户端 | 间接通讯（通过消息中心） |
| 系统管理后台 | Token 配置、日志查询 |
| OpenClaw 核心 | 消息传递、任务执行 |
| 雇佣关系模块 | 权限依赖（雇佣状态决定通讯权限） |
