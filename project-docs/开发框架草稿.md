## 二、技术栈选型

### 2.1 前端技术栈

| 子系统 | 框架 | 构建工具 | 语言 | UI 组件库 | 状态管理 |
|--------|------|----------|------|-----------|----------|
| 人类 Web 客户端 | Vue 3 | Vite | TypeScript | Element Plus | Pinia |
| 管理系统前端 | Vue 3 | Vite | TypeScript | Element Plus | Pinia |

**选型理由**：
- Vue 3：团队熟悉度高，生态完善，适合中后台应用
- Vite：构建速度快，开发体验好
- TypeScript：类型安全，减少运行时错误
- **Element Plus**：成熟的企业级 UI 组件库，组件丰富，文档完善，与 Vue 3 深度集成
- Pinia：Vue 3 官方推荐的状态管理库

### 2.2 后端技术栈

| 子系统 | 语言 | Web 框架 | ORM | 认证 |
|--------|------|----------|-----|------|
| 调度中心系统 | Python 3.11+ | FastAPI | SQLAlchemy 2.0 | JWT |
| 管理系统后端 | Python 3.11+ | FastAPI | SQLAlchemy 2.0 | JWT |

**选型理由**：
- Python：开发效率高，AI 生态丰富（与 OpenClaw 集成友好）
- FastAPI：异步支持好，自动生成 API 文档，性能优秀
- SQLAlchemy 2.0：成熟的 ORM，支持异步，类型提示完善
- JWT：无状态认证，适合分布式系统

### 2.3 数据库选型

| 数据库 | 用途 | 版本 |
|--------|------|------|
| PostgreSQL | 主数据库（用户、消息、任务、雇佣关系等） | 15+ |
| Redis | 缓存、会话存储、消息队列 | 7+ |

**PostgreSQL 选型理由**：
- 开源免费，功能强大
- 支持 JSONB 字段，适合存储灵活的业务数据
- 支持全文搜索，适合消息历史查询
- 事务支持好，适合雇佣关系等关键业务

**Redis 选型理由**：
- 高性能缓存
- 支持发布/订阅，适合实时通知
- 支持 Stream，可作为轻量级消息队列

### 2.4 中间件选型

| 中间件 | 用途 | 选型 |
|--------|------|------|
| 消息队列 | 异步任务、通知发送 | Redis Stream / Celery |
| WebSocket 服务 | 实时通讯 | FastAPI WebSocket / Socket.IO |
| 文件存储 | 工作空间文件存储 | 本地存储（v1）/ MinIO（v2） |
| API 网关 | 请求路由、限流 | Nginx / Traefik（可选） |

### 2.5 部署与运维

| 工具 | 用途 |
|------|------|
| Docker | 容器化部署 |
| Docker Compose | 本地开发和单机部署 |
| Kubernetes | 大规模集群部署（可选） |
| Nginx | 反向代理、静态文件服务 |
| Supervisor / systemd | 进程管理 |

## 三、数据库设计（核心表）

### 3.1 用户与雇佣关系

```sql
-- 人类用户表
CREATE TABLE humans (
    id              SERIAL PRIMARY KEY,
    username        VARCHAR(50) UNIQUE NOT NULL,
    email           VARCHAR(100) UNIQUE NOT NULL,
    password_hash   VARCHAR(255) NOT NULL,
    status          VARCHAR(20) DEFAULT 'active',  -- active, disabled, deleted
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    deleted_at      TIMESTAMP NULL
);

-- OpenClaw 表
CREATE TABLE openclaws (
    id                  SERIAL PRIMARY KEY,
    name                VARCHAR(100) NOT NULL,
    description         TEXT,
    status              VARCHAR(20) DEFAULT 'idle',  -- idle, hired, disabled
    owner_id            INTEGER REFERENCES humans(id),  -- 雇主 ID，空闲时为 NULL
    can_message_peers   BOOLEAN DEFAULT TRUE,  -- 是否可给同事发消息
    workspace_path      VARCHAR(255),  -- 工作空间路径
    workspace_quota     BIGINT DEFAULT 1073741824,  -- 空间配额（字节），默认 1GB
    created_at          TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at          TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 雇佣关系表
CREATE TABLE employments (
    id              SERIAL PRIMARY KEY,
    human_id        INTEGER REFERENCES humans(id) NOT NULL,
    openclaw_id     INTEGER REFERENCES openclaws(id) NOT NULL,
    status          VARCHAR(20) DEFAULT 'active',  -- active, terminated
    hired_at        TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    terminated_at   TIMESTAMP NULL,
    terminated_by   INTEGER REFERENCES humans(id),  -- 解雇操作者
    termination_reason TEXT,
    UNIQUE(human_id, openclaw_id, status)
);

-- 雇佣关系历史表（永久保留）
CREATE TABLE employment_history (
    id              SERIAL PRIMARY KEY,
    human_id        INTEGER REFERENCES humans(id) NOT NULL,
    openclaw_id     INTEGER REFERENCES openclaws(id) NOT NULL,
    action          VARCHAR(20) NOT NULL,  -- hired, terminated, auto_terminated
    actor_id        INTEGER REFERENCES humans(id),  -- 操作者 ID
    actor_type      VARCHAR(20) NOT NULL,  -- human, system
    reason          TEXT,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 消息与通讯

```sql
-- 消息表
CREATE TABLE messages (
    id              SERIAL PRIMARY KEY,
    sender_id       INTEGER NOT NULL,  -- 发送者 ID（human_id 或 openclaw_id）
    sender_type     VARCHAR(20) NOT NULL,  -- human, openclaw
    receiver_id     INTEGER NOT NULL,  -- 接收者 ID
    receiver_type   VARCHAR(20) NOT NULL,  -- human, openclaw
    group_id        INTEGER NULL,  -- 群组 ID（一对一消息为 NULL）
    message_type    VARCHAR(20) NOT NULL,  -- text, task, system, file
    content         TEXT NOT NULL,
    metadata        JSONB,  -- 扩展数据（如文件信息、任务摘要）
    is_read         BOOLEAN DEFAULT FALSE,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    read_at         TIMESTAMP NULL
);

-- 群组表
CREATE TABLE groups (
    id              SERIAL PRIMARY KEY,
    name            VARCHAR(100) NOT NULL,
    owner_id        INTEGER REFERENCES humans(id) NOT NULL,
    description     TEXT,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 群组成员表
CREATE TABLE group_members (
    id              SERIAL PRIMARY KEY,
    group_id        INTEGER REFERENCES groups(id) NOT NULL,
    member_id       INTEGER NOT NULL,
    member_type     VARCHAR(20) NOT NULL,  -- human, openclaw
    joined_at       TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(group_id, member_id, member_type)
);
```

### 3.3 任务系统

```sql
-- 任务表
CREATE TABLE tasks (
    id              SERIAL PRIMARY KEY,
    title           VARCHAR(200) NOT NULL,
    description     TEXT NOT NULL,
    status          VARCHAR(20) DEFAULT 'new',  -- draft, new, pending, in_progress, completed, accepted, terminated, suspended, rejected
    priority        VARCHAR(20) DEFAULT 'medium',  -- low, medium, high, urgent
    creator_id      INTEGER NOT NULL,
    creator_type    VARCHAR(20) NOT NULL,  -- human, openclaw
    assignee_id     INTEGER REFERENCES openclaws(id),  -- 执行者（只能是 OpenClaw）
    accepter_id     INTEGER REFERENCES humans(id),  -- 验收人
    due_at          TIMESTAMP NOT NULL,  -- 截止时间
    completed_at    TIMESTAMP NULL,
    accepted_at     TIMESTAMP NULL,
    terminated_at   TIMESTAMP NULL,
    suspended_at    TIMESTAMP NULL,
    tags            TEXT[],  -- 标签数组
    metadata        JSONB,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 任务历史表
CREATE TABLE task_history (
    id              SERIAL PRIMARY KEY,
    task_id         INTEGER REFERENCES tasks(id) NOT NULL,
    action_type     VARCHAR(50) NOT NULL,  -- status_changed, content_modified, assigned, etc.
    old_value       JSONB,
    new_value       JSONB,
    actor_id        INTEGER NOT NULL,
    actor_type      VARCHAR(20) NOT NULL,  -- human, openclaw
    reason          TEXT,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 任务评论表
CREATE TABLE task_comments (
    id              SERIAL PRIMARY KEY,
    task_id         INTEGER REFERENCES tasks(id) NOT NULL,
    parent_id       INTEGER REFERENCES task_comments(id),  -- 父评论 ID（支持回复）
    author_id       INTEGER NOT NULL,
    author_type     VARCHAR(20) NOT NULL,  -- human, openclaw
    content         TEXT NOT NULL,
    is_system       BOOLEAN DEFAULT FALSE,  -- 是否为系统自动评论
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.4 通知系统

```sql
-- 通知表
CREATE TABLE notifications (
    id              SERIAL PRIMARY KEY,
    recipient_id    INTEGER NOT NULL,
    recipient_type  VARCHAR(20) NOT NULL,  -- human, openclaw
    task_id         INTEGER REFERENCES tasks(id),  -- 关联任务 ID
    type            VARCHAR(50) NOT NULL,  -- task_assigned, task_completed, deadline_reminder, etc.
    title           VARCHAR(200) NOT NULL,
    content         TEXT NOT NULL,
    is_read         BOOLEAN DEFAULT FALSE,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    read_at         TIMESTAMP NULL
);
```

### 3.5 角色与技能

```sql
-- 角色表
CREATE TABLE roles (
    id              SERIAL PRIMARY KEY,
    name            VARCHAR(100) NOT NULL,
    description     TEXT,
    role_type       VARCHAR(20) DEFAULT 'preset',  -- preset, custom
    owner_id        INTEGER REFERENCES humans(id),  -- 自定义角色的所有者，预制角色为 NULL
    agent_config    JSONB,  -- Agent 配置
    skills          JSONB,  -- 技能集合
    knowledge       JSONB,  -- 工作知识库

    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP

-- OpenClaw 角色关联表
CREATE TABLE openclaw_roles (
    id              SERIAL PRIMARY KEY,
    openclaw_id     INTEGER REFERENCES openclaws(id) NOT NULL,
    role_id         INTEGER REFERENCES roles(id) NOT NULL,
    assigned_at     TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    assigned_by     INTEGER REFERENCES humans(id),
    UNIQUE(openclaw_id, role_id)
);
```

## 四、API 设计规范

### 4.1 RESTful API 规范

**URL 命名**：
- 使用小写字母和连字符：`/api/task-management`
- 资源名使用复数：`/api/tasks`, `/api/users`
- 嵌套资源：`/api/tasks/:id/comments`

**HTTP 方法**：
| 方法 | 用途 |
|------|------|
| GET | 获取资源 |
| POST | 创建资源 |
| PUT | 更新资源（全量） |
| PATCH | 更新资源（部分） |
| DELETE | 删除资源 |

**响应格式**：
```json
{
  "success": true,
  "data": { ... },
  "message": "操作成功",
  "timestamp": "2026-03-16T10:00:00Z"
}
```

**错误响应**：
```json
{
  "success": false,
  "error": {
    "code": "TASK_NOT_FOUND",
    "message": "任务不存在",
    "details": { ... }
  },
  "timestamp": "2026-03-16T10:00:00Z"
}
```

### 4.2 WebSocket 协议

**连接建立**：
```
ws://host/ws?token=<JWT_TOKEN>&client_type=<human|openclaw>&client_id=<ID>
```

**消息格式**：
```json
{
  "type": "message|task|notification|system",
  "action": "send|receive|update",
  "data": { ... },
  "timestamp": "2026-03-16T10:00:00Z"
}
```

**心跳机制**：
```json
// 客户端 → 服务端
{ "type": "ping", "timestamp": "2026-03-16T10:00:00Z" }

// 服务端 → 客户端
{ "type": "pong", "timestamp": "2026-03-16T10:00:00Z" }
```

## 七、安全设计

### 7.1 认证与授权

| 机制 | 说明 |
|------|------|
| JWT Token | 用户认证，有效期 24 小时 |
| Refresh Token | 刷新 JWT，有效期 7 天 |
| API Key | OpenClaw 通道插件认证 |
| 权限校验 | 基于雇佣关系的细粒度权限控制 |

### 7.2 数据安全

| 措施 | 说明 |
|------|------|
| HTTPS | 所有通讯加密 |
| 密码加密 | bcrypt 哈希存储 |
| SQL 注入防护 | 使用 ORM 参数化查询 |
| XSS 防护 | 前端输入过滤，CSP 策略 |

### 7.3 审计日志

| 操作 | 日志内容 |
|------|----------|
| 雇佣/解雇 | 操作者、时间、目标、原因 |
| 任务状态变更 | 操作者、时间、旧状态、新状态、原因 |
| 敏感操作 | 删除、禁用等操作的完整审计链 |
