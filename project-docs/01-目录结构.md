# 1Claw 目录结构

本文档描述 1Claw 项目的目录结构及各模块职责。

---

## 目录结构

```
1claw/
├── channel/                 # OpenClaw 通道插件
├── scheduler-server/        # 调度中心（后端服务）
├── project-docs/            # 项目文档
└── web-client/              # Web 客户端（用户界面）
```

---

## 各目录说明

### 1. `channel/` — OpenClaw 通道插件

**定位**：安装于 OpenClaw 端的专属插件

**职责**：
- 实现 OpenClaw 与调度中心的双向通讯
- 承接 OpenClaw 的消息接收与指令反馈

**技术栈**：OpenClaw Plugin

---

### 2. `scheduler-server/` — 调度中心

**定位**：全系统调度中枢（纯后端）

**职责**：
- OpenClaw 管理（注册、启用/禁用）
- 消息收发、分发
- 任务管理（创建、分配、追踪、通知）
- 截止时间提醒与超期处理

**技术栈**：
- 后端：Python + SQLite
- 接口：RESTful API + WebSocket

---

### 3. `web-client/` — Web 客户端

**定位**：面向个人用户的 Web 端交互界面

**职责**：
- OpenClaw 管理（添加、启用/禁用、配置）
- 通讯交互（一对一对话、群组对话）
- 任务分配与管理

**技术栈**：
- 前端：Vue + Vite + TypeScript

---

### 4. `project-docs/` — 项目文档

**定位**：项目过程文档存放目录

**规范**：
- 文档格式：Markdown
- 文件命名：`.md` 结尾

**现有文档**：
```
project-docs/
├── README.md                # 文档索引
├── development/             # 开发指南
│   ├── README.md
│   └── 01-开发框架.md
├── 00-需求概述.md           # 软件定位
├── 01-目录结构.md           # 本文档
├── 03-Agent 设计.md          # Agent 设计
├── 11-人类 Web 客户端.md    # Web 客户端设计
├── 12-1Claw OpenClaw 通道插件.md
├── 13-调度中心系统.md       # 调度中心架构
├── 13-1-任务管理.md         # 任务管理需求
└── 20-系统架构.md           # 系统架构设计
```

---

## 开发规范

1. **文档更新**：所有过程文档写入 `project-docs/` 目录
2. **代码组织**：各模块保持独立，通过调度中心 API 通讯
3. **前后端分离**：前端工程与后端工程分目录管理
4. **版本控制**：统一使用 Git 管理，提交信息清晰描述变更
