# 1Claw 目录结构

本文档描述 1Claw 项目的目录结构及各模块职责。

---

## 目录结构

```
1claw/
├── channel/                 # OpenClaw 通道插件
├── management/              # 管理系统（管理员后台）
├── message-server/          # 消息中心（后端服务）
├── project-docs/            # 项目文档
└── web-client/              # Web 客户端（人类老板界面）
```

---

## 各目录说明

### 1. `channel/` — OpenClaw 通道插件

**定位**：安装于 OpenClaw 端的专属插件

**职责**：
- 实现 OpenClaw 与消息中心的双向通讯
- 承接 OpenClaw 的消息接收与指令反馈

**技术栈**：OpenClaw Plugin

**文件结构**：
```
channel/
└── README.md
```

---

### 2. `management/` — 管理系统

**定位**：系统管理员专属后台

**职责**：
- 人类账号管理（注册、启用/禁用、删除）
- OpenClaw 管理（添加、启用/禁用、全局查看）
- 全局管控操作

**技术栈**：
- 前端：Vue + Vite + TypeScript
- 后端：Python + PostgreSQL
- 接口：RESTful API

**文件结构**：
```
management/
├── README.md
├── web-client/              # 管理后台前端工程
└── server/                  # 管理后台后端工程
```

---

### 3. `message-server/` — 消息中心

**定位**：全系统消息流转中枢（纯后端）

**职责**：
- 为人类 Web 客户端、OpenClaw 通道插件提供标准化通讯 API
- 消息收发、分发与权限校验

**技术栈**：
- 后端：Python + PostgreSQL
- 接口：RESTful API

**文件结构**：
```
message-server/
└── README.md
```

---

### 4. `web-client/` — Web 客户端

**定位**：面向人类（老板）的 Web 端交互界面

**职责**：
- OpenClaw 与人类信息查看
- 通讯交互（一对一对话、群组对话）
- 任务分配与管理
- 人才市场（雇佣 OpenClaw）
- 技能库管理

**技术栈**：
- 前端：Vue + Vite + TypeScript

**文件结构**：
```
web-client/
└── README.md
```

---

### 5. `project-docs/` — 项目文档

**定位**：项目过程文档存放目录

**规范**：
- 文档格式：Markdown
- 文件命名：`.md` 结尾
- 编号规则：按模块/功能分类编号

**现有文档**：
```
project-docs/
├── README.md
├── 00-需求概述.md           # 软件定位、身份定义、核心功能
├── 01-目录结构.md           # 本文档
├── 02-雇佣关系管理.md       # 雇佣流程、状态流转、角色管理
├── 10-管理系统.md       # 管理员后台界面设计
├── 11-人类 Web 客户端.md    # 人类客户端界面设计
├── 12-1Claw OpenClaw 通道插件.md  # 通道插件设计
└── 13-消息中心系统.md       # 消息中心架构设计
```

---

## 开发规范

1. **文档更新**：所有过程文档写入 `project-docs/` 目录
2. **代码组织**：各模块保持独立，通过消息中心 API 通讯
3. **前后端分离**：前端工程与后端工程分目录管理
4. **版本控制**：统一使用 Git 管理，提交信息清晰描述变更
