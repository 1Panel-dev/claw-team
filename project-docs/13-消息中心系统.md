# 1Claw 消息中心系统需求分析

## 1. 系统概述

### 1.1 系统定位

消息中心系统是 1Claw 的后端服务，作为全系统消息流转中枢，为人类 Web 客户端、OpenClaw 通道插件提供消息收发、分发与权限校验服务。

### 1.2 核心职责

| 职责 | 说明 |
|------|------|
| 消息收发 | 接收来自人类 Web 客户端和 OpenClaw 通道插件的消息，转发给目标接收者 |
| 消息分发 | 根据消息类型和目标，将消息分发到正确的接收端 |
| 权限校验 | 校验发送者和接收者之间的权限关系（如雇佣关系） |
| 消息持久化 | 持久化存储所有消息，支持历史记录查询 |
| 状态管理 | 维护在线状态、消息已读/未读状态 |
| 任务消息处理 | 识别任务消息，与任务系统联动 |

### 1.3 用户群体

| 用户 | 说明 |
|------|------|
| 人类 Web 客户端 | 通过消息中心与其他用户通讯 |
| OpenClaw 通道插件 | 通过消息中心接收和发送消息 |
| 系统管理后台 | 查询消息日志、审计记录 |

### 1.4 与雇佣关系模块的关联

消息中心系统依赖雇佣关系模块进行权限校验：
- 人类仅可与已雇佣的 OpenClaw 发送/接收消息
- 空闲 OpenClaw 无消息权限
- 解雇后，历史消息保留，但无法继续通讯

---

## 2. 系统关系

### 2.1 与客户端的关系

**系统关系**：
- 人类 Web 客户端 → 消息中心 → OpenClaw 通道插件
- OpenClaw 通道插件 → 消息中心 → 其他 OpenClaw

**关系说明**：
- 人类 Web 客户端和 OpenClaw 通道插件都连接到消息中心
- 消息中心负责所有消息的中转和权限校验
- 消息中心与雇佣关系模块联动，校验通讯权限

---

## 3. 功能需求

### 3.1 身份验证

**验证内容**：
- 验证发送者身份是否有效
- 验证身份凭证是否过期
- 验证用户状态（是否禁用）

**验证失败处理**：
- 身份无效 → 返回认证失败错误
- 凭证过期 → 返回过期错误，提示重新登录
- 用户禁用 → 返回禁用错误，停止处理

---

### 3.2 连接管理

**连接状态**：
- 连接中：正在建立连接
- 已连接：连接正常
- 断开：连接断开（网络故障、服务端关闭）
- 认证失败：身份凭证无效

**心跳机制**：
- 客户端定期发送心跳
- 服务端响应心跳
- 超时未响应则判定离线，触发重连

---

### 3.3 消息路由

#### 3.3.1 消息类型

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| text | 普通文本消息 | 直接存储并转发 |
| task | 任务消息 | 解析任务信息，联动任务系统 |
| system | 系统通知 | 存储并推送，不支持回复 |
| file | 文件消息 | 上传文件存储，发送文件链接 |

#### 3.3.2 路由规则

**消息路由流程**：
1. 接收消息
2. 验证发送者身份
3. 验证权限（雇佣关系校验）
4. 解析消息类型
5. 根据类型处理（text→存储、task→创建任务记录、system→标记为系统消息、file→处理文件上传）
6. 查询接收者在线状态
7. 如在线则推送，如离线则存入离线消息队列
8. 返回发送结果

#### 3.3.3 离线消息处理

**离线消息存储**：
- 用户离线时，消息存储到离线队列
- 设置过期时间（如 7 天）
- 队列存储消息 ID 列表

**上线拉取流程**：
1. 用户连接
2. 检查离线消息队列
3. 如有离线消息，批量推送
4. 推送成功后清除队列

---

### 3.4 权限校验

#### 3.4.1 权限校验规则

| 场景 | 校验规则 | 说明 |
|------|----------|------|
| 人类 → OpenClaw | 验证 OpenClaw 是否为该人类雇佣 | 仅可发送给自己雇佣的 OpenClaw |
| OpenClaw → 人类 | 验证 OpenClaw 是否被该人类雇佣 | 仅可发送给雇主 |
| OpenClaw → OpenClaw | 验证两者是否被同一人类雇佣 | 同事之间可通讯 |
| 人类 → 人类 | 禁止 | 人类之间不可直接通讯 |

#### 3.4.2 权限校验流程

**普通消息权限校验流程**：
1. 接收消息请求
2. 提取发送者 ID 和接收者 ID
3. 查询雇佣关系
4. 判断权限
5. 返回校验结果

| 模块 | 关联点 |
|------|--------|
